操作部分

“操作”部分包含指定每条指令执行语法的代码。它不包含与指令获取有关的任何行为（例如，获取期间导致的中断与错误）。

要查看“操作”部分，请单击任何指令标题中“操作”字样。

这部分的字体惯例是：

代码 -（“操作”部分）指定指令行为的 C 代码。

代码_斜体 -（“操作”部分）对应“格式”部分列出的斜体字段的汇编语言字段名。

代码惯例

在“操作”部分，寄存器使用表示法 reg[addr].field 寻址。访问的寄存器文件由 reg 指定，它的值从寄存器文件表示法表格的第二列中选择。addr 字段按照汇编语言字段名或寄存器助记符的形式指定寄存器地址。field 选项指定寄存器内命名的位字段。如果 field 不存在，则访问寄存器的所有字段。唯一的例外是引用通用寄存器（64 位，不含 NaT）的数据字段时，该处使用表示法 GR[addr]。“操作”部分中的代码与标准 C 之间的语法差异列在 C 代码与操作部分代码的语法差异表格中。

由于传统编程语言语句的顺序性质，用于指定指令操作的代码会在具体实现的版本上施加一些人为的限制。英特尔(R) 安腾(R) 处理器可能会提前主动去执行一些操作，后来却确定并不需要这些操作的结果，因此随后会抑制其体系结构上可见状态的更新。为体现这种自由度，对于“操作”部分出现的代码，适用以下规则：

  1. 赋值语句右侧指定的操作可独立于前面的任何 "if" 控制语句执行。如果赋值语句的右侧包含过程调用，则任何过程参数的求值都可独立于前面的任何 "if" 控制语句。同样的规则也递归适用于过程本身及其它衍生物中的语句。在后面的指令说明中，过程参数的表达式从不会有体系结构上可见的副作用（例如，对体系结构状态的赋值）。

  2. 如果赋值语句左侧的变量代表体系结构上可见的状态，除非前面所有的 "if" 控制语句的值为真，否则不发生赋值。如果赋值语句左侧的变量不代表体系结构上可见的状态（例如，临时变量），则赋值可独立于前面的任何 "if" 控制语句发生。如果语句仅由一个过程调用组成，则相同的规则递归适用于过程本身及其衍生物中的语句。

  3. "if" 语句的条件表达式可独立于前面的任何 "if" 控制语句的结果求值。在后面的指令说明中，"if" 语句的条件表达式从不会有体系结构上可见的副作用（例如，对体系结构状态的赋值）。

在伪代码示例中，根据第一项规则：

  • tlb_translate() 可独立于 PR[qp] 的值与 aligned() 返回的值进行调用

  • mem_read() 与 zero_ext() 可独立于 PR[qp] 与 defer 的值调用

  • GR[r3]+GR[r2] 操作可独立于 PR[qp]、defer 及 register_form 的值执行

根据第二项规则：

  • 除非 PR[qp] 为真，defer 为假，否则 GR[r1] 不会更新

  • 除非 PR[qp] 为真，defer 为假，并且 register_form 为真，否则 GR[r3] 不会更新

  • 除非 PR[qp] 为真，defer 为假，否则不会调用 alat_inval_single_entry()

  • 除非 PR[qp] 为真，aligned() 返回假，否则不会调用 unaligned_data_reference_fault()

  • tmp_paddr 可独立于 PR[qp] 的值进行更新

根据第三项规则：

  • aligned() 可独立于 PR[qp] 的值进行调用

操作部分中的变量

“操作”部分的代码中出现的所有变量都是“通用数据类型”，是可变长无符号位字段。代表体系结构状态的变量使用固定的字段长度，与它们的定义保持一致。临时变量（即，非体系结构变量）默认使用最新赋值语句右侧最长变量的长度。

如果赋值语句右侧的变量比左侧的变量长，则在赋值之前，代表右侧变量的位字段将截断至左侧变量的长度。如果赋值语句右侧的变量比左侧的变量短，则在赋值之前，代表右侧变量的位字段将按照零扩展方式扩展至左侧变量的长度。这些规则同样适用于在不同长度的变量上执行的操作。
