中断返回指令

rfi

操作 字体惯例

格式

rfi        B8

说明

还原中断前的机器上下文。PSR 从 IPSR 还原，IPSR 保持不变，IP 从 IIP 还原。执行从加载到 IP 的指令束地址与 PSR.ri 中加载的指令槽处继续。

此指令必须紧跟在停止后面。否则会发生“非法操作”错误。此指令切换到 IPSR.bn 指定的寄存器库。相同指令组中访问 GR16 到 GR31 的指令引用前一个寄存器库。后续指令组引用新的寄存器库。

此指令执行指令序列化以确保：

  • 观察到先前对处理器寄存器资源的修改，这些资源影响后续指令组的获取。

  • 观察到先前对处理器寄存器资源的修改，这些资源影响后续的执行或数据内存访问。

  • 先前的内存同步 (sync.i) 操作开始在本地处理器指令缓存上起作用。

  • 在 rfi 完成之后，已重新初始化后续的指令组获取（包括目标指令组）。

rfi 指令必须在包含要序列化的操作的指令组后面的指令组中。

此指令只能在最高特权级别执行。此指令无法断定。

如果 PSR.ic 或 PSR.i 为 1，则此指令的执行未定义。软件必须确保，在写入 IIP、IPSR 或 IFS 与后续的 rfi 之间，不能发生可能会修改这些寄存器的中断。

此指令不执行“低特权传输”、“执行分支”或“单步”陷阱。

如果目标是包含 movl 指令的指令束，并且此指令将 PSR.ri 设为 2，则目标指令束上产生“非法操作”错误。

如果 IPSR.is 为 1，则在 IIP{31:0} 指定的虚拟线性地址上恢复 IA-32 指令集的控制。PSR.di 不会阻止此指令的指令集转换。在 rfi 完成执行之后，如果 PSR.dfh 为 1，则在目标 IA-32指令上产生“禁用的 FP 寄存器”错误。

如果 IPSR.is 为 1，并发生“未使用的指令地址”陷阱，则 IIP 将包含原始的 64 位目标 IP。（此值不是从 32 位通过零扩展方式得到的）。

进入 IA-32 指令集时，当前堆栈帧的大小会设置为零，且堆栈中的所有通用寄存器都处于未定义状态。在指令集转换过程中，软件不能依赖这些寄存器的值。软件必须确保在进入 IA-32 指令集时 AR[BSPSTORE]==AR[BSP]，否则可能会导致未定义的行为。

如果 IPSR.is 是 1，则软件必须为执行 IA-32 指令集正确设置其它的 IPSR 字段；否则处理器操作未定义。

如需有关详细信息，请参阅 Intel(R) Itanium(R) Architecture Software Developer’s Manual（英特尔(R) 安腾(R) 体系结构软件开发人员手册）第 2 卷第 2:19 页上的表 3-2“处理器状态寄存器字段”。

在 IA-32 处理器的一致内存引用与安腾(R) 体系结构的无序内存引用之间，如果需要确定内存顺序，则软件必须在此条指令之前发出 mf 指令。处理器不确保后续的 IA-32 指令能看见安腾指令集生成的指令流写入。

软件必须确保在发出此指令之前，代码段描述符与选择器已经加载。如果目标 EIP 值超出代码段的限制，或是存在代码段特权冲突，则在 IA-32 指令上生成 IA-32_Exception(GPFault) 异常。进入 16 位 IA-32 代码时，如果 IIP 不在 CSD.base 的 64K 字节范围内，则在目标指令上生成 GPFault。

直到目标 IA-32 指令成功完成之后，才会修改 EFLAG.rf 与 PSR.id。在目标 IA-32 指令开始执行之前，PSR.da、PSR.dd、PSR.ia 及 PSR.ed 都清除为零。

执行 IA-32 指令集时，ALAT 的内容保持未定义。在指令集转换过程中，软件不能依赖 ALAT 状态。进入 IA-32 代码时，将忽略 ALAT 中现有的项目。

中断

  • “非法操作”错误；“特权操作”错误

  • “未使用的指令地址”陷阱

  • IA-32 目标指令上的其它错误

    • IA-32_Exception(GPFault)

    • 禁用的 FP Reg 错误 - 如果 PSR.dfh 是 1
