加载指令

ld

操作 字体惯例

指令类型 M

格式

(qp) ldsz.ldtype.ldhint r1 = [r3] no_base_update_form

(qp) ldsz.ldtype.ldhint r1 = [r3], r2 reg_base_update_form

(qp) ldsz.ldtype.ldhint r1 = [r3], imm9 imm_base_update_form

(qp) ld8.fill.ldhint r1 = [r3] fill_form, no_base_update_form

(qp) ld8.fill.ldhint r1 = [r3], r2 fill_form, reg_base_update_form

(qp) ld8.fill.ldhint r1 = [r3], imm9 fill_form, imm_base_update_form

说明

从 GRr3 中的值指定的地址开始，从内存中读取 sz 个字节组成的值。然后此值执行零扩展，并放入 GR r1。清除 GR r1 对应的 NaT 位，下面在推测性加载部分介绍的除外。ldtype 完成符指定特殊的加载操作。

对于 fill_form，加载 8 字节值，UNAT 应用寄存器中的位复制到目标寄存器 NaT 位。此指令用于重新加载转存的寄存器/NaT 对。

在基址更新形式中，GR r3 中的值加到有符号的立即数 (imm9) 或 GR r2 中的值上，随后结果放回 GR r3。加载之后便完成此基址寄存器的更新，且不影响加载地址。在 reg_base_update_form 中，如果设置 GR r2 对应的 NaT 位，则 GR r3 对应的 NaT 位也会设置为 1，且不触发任何错误。

对于非推测性加载类型，如果与 GR r3 关联的 NaT 位是 1，则执行“寄存器 NaT 使用”错误的处理。对于推测性加载与推测性高级加载，不触发任何错误，而且会缓发异常。对于基址更新计算，如果与 GR r2 关联的 NaT 位是 1，则与 GR r3 关联的 NaT 位设置为 1，而且不触发任何错误。

ldhint 完成符的值指定内存访问的区域性。在基址更新形式中，会暗示预取提示。基址更新之后 GR r3 中的值指定的地址充当预取指定的缓存线提示。此预取操作使用 ldhint 指定的区域性提示。预取与区域性提示不影响程序功能，在具体实现的版本中可能会被忽略。

在 no_base_update 形式中，GR r3 中的值不修改，并且不暗示任何预取提示。

对于基址更新形式，在 r1 与 r3 中指定相同的寄存器地址时，将会导致“非法操作”错误。
