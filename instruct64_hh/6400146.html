分支指令

br

操作 字体惯例

指令类型 B

格式

(bp) br.btype.bwh.ph.dh target25 IP_relative_form

(bp) br.btype.bwh.ph.dh b1 = target25 call_form, IP_relative_form

br.btype.bwh.ph.dh target25 counted_form, IP_relative_form

br.ph.dh target25 pseudo-op

(bp) br.btype.bwh.ph.dh b2 indirect_form

(bp) br.btype.bwh.ph.dh b1 = b2 call_form, indirect_form

br.ph.dh b2 pseudo-op

说明

对分支条件求值，根据结果执行分支，或继续执行序列中的下一条指令。从逻辑意义上说，分支的执行跟在相同指令组中前面所有非分支指令的执行之后。在执行的分支上，执行从指令槽 0 开始。

分支可以与 IP 相对分支或间接分支。对于 IP 相对分支，汇编中的 target25 操作数指定要执行的分支的标签。在分支指令中，它作为有符号立即数位移量 (imm21) 进行编码，放在目标分支束与包含此指令的分支束之间 (imm21 = target25 - IP >> 4)。对于间接分支，目标地址取自 BR b2。

无条件分支有两种伪操作。它们的编码与条件分支类似 (btype = COND)，bp 字段指定 PR 0，并带 bwh 提示 sptk。

基本分支类型

分支类型确定如何计算分支条件，以及分支是否有其它效果（如写入链接寄存器）。对于基本分支类型，分支条件只是指定的谓词寄存器中的值（称为分支谓词）。

基本分支类型包括：

  • cond

  • call

  • return

  • ia

  • loops

对于分支指令，“先写后读”(RAW) 与“先读后写”(WAR) 在相关性要求方面略有不同。对于相同指令组中的后续分支指令而言，非分支指令对 BR、PR（谓词寄存器）及 PFS 的更改都是可见的，也就是说，这些资源可使用受限制的 RAW。譬如说，低延迟的比较分支序列就是如此。常规 RAW 要求适用于 LC、EC 应用寄存器以及 RRB。

在指令组中，如果读取与写入指令都出现分支，则不允许在 PR 63 上存在 WAR 相关性。例如，br.wtop 或 br.wexit 不得使用 PR[63] 作为其限定谓词，并且 PR[63] 不得是相同指令组中 br.wtop 或 br.wexit 之前任何分支的限定谓词。

出于相关性方面的目的，循环型分支总是会有效地写入关联的资源，无论它们是否执行。cloop 类型总是会有效地写入 LC。LC 为 0 时，cloop 分支让它保持不变，但硬件可能会按照使用相同的值重写 LC 的办法来实施这种行为。与此类似，br.ctop 与 br.cexit 总是有效地写入 LC、EC、RRB 以及 PR[63]；br.wtop 与 br.wexit 总是有效地写入 EC、RRB 以及 PR[63]。
