比较字指令

cmp4

操作 字体惯例

指令类型 A

格式

(qp) cmp4.crel.ctype p1, p2 = r2, r3 register_form

(qp) cmp4.crel.ctype p1, p2 = imm8, r3 imm8_form

(qp) cmp4.crel.ctype p1, p2 = r0, r3 parallel_inequality_form

(qp) cmp4.crel.ctype p1, p2 = r3, r0 pseudo-op

说明

按照 crel 指定的十种关系之一，比较两个源操作数中的最低有效 32 位。如果比较条件为真，则产生的布尔结果为 1，反之则为 0。此结果写入两个谓词寄存器目的地 p1 与 p2。结果写入目的地的方式由 ctype 指定的比较类型确定，如比较 (cmp) 指令中所述。

在 register_form 中，第一个操作数是 GR r2；在 imm8_form 中，第一个操作数取自符号扩展的 imm8 编码字段；在 parallel_inequality_form 中，第一个操作数必须是 GR 0。

只有比较类型是并行类型之一，且关系是不等式（>、>=、<、<=）时，才会使用 parallel_inequality_form，如比较 (cmp) 指令与并行类型比较的 64 位比较关系表格所述。

如果两个谓词寄存器目的地相同（p1 与 p2 指定相同的谓词寄存器）、限定谓词已设置或比较类型 (ctype) 为 unc，则指令都将发生“非法操作”错误。

在这十种关系中，并非所有都直接在硬件中实现。有一些实际上是伪操作。实现的关系以及这些伪操作如何映射到它们的信息详见标准与 UNC 类型比较的 64 位比较关系与并行类型比较的 64 位比较关系。
