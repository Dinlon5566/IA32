比较指令

cmp

操作 字体惯例

指令类型 A

格式

(qp) cmp.crel.ctype p1, p2 = r2, r3 register_form

(qp) cmp.crel.ctype p1, p2 = imm8, r3 imm8_form

(qp) cmp.crel.ctype p1, p2 = r0, r3 parallel_inequality_form

(qp) cmp.crel.ctype p1, p2 = r3, r0 pseudo-op

说明

按照 crel 指定的十种关系之一比较两个源操作数。如果比较条件为真，则产生的布尔结果为 1，反之则为 0。此结果写入两个谓词寄存器目的地 p1 与 p2。结果写入目的地的方式由 ctype 指定的比较类型确定。

比较类型描述如何随根据比较结果更新谓词目标。常规类型仅将比较结果写入一个目标，而在另一个目标中写入相反的内容。并行类型只更新特定比较结果的目标。这样便可以针对相同的谓词寄存器同时进行多个“或”型或多个“与”型比较。

unc 首先将谓词目标初始化为 0，而不管限定谓词如何，因此比较特殊。然后它与常规类型的行为方式相同。比较类型的行为在比较类型表格中介绍。。空白项目表示谓词目标保持不变。

在 register_form 中，第一个操作数是 GR r2；在 imm8_form 中，第一个操作数取自符号扩展的 imm8 编码字段；在 parallel_inequality_form 中，第一个操作数必须是 GR 0。只有比较类型是并行类型之一，且关系是不等式（>、>=、<、<=）时，才会使用 parallel_inequality_form。请参阅下文。

如果两个谓词寄存器目的地相同（p1 与 p2 指定相同的谓词寄存器）、限定谓词已设置或比较类型为 unc，则指令都将执行“非法操作”错误的处理。

在这十种关系中，并非所有都直接在硬件中实现。有一些实际上是伪操作。对于这些情况，汇编器只是切换一下源操作数指示符且/或切换谓词目标指示符，然后使用实现的关系。对于 imm8_form 形式的一些伪操作比较，汇编器从立即数中减 1，使得允许的立即数范围略有不同。在六种并行比较类型中，有三种类型实际上是伪操作。汇编器只是使用与实现的类型相反的关系。具体实现的关系以及这些伪操作与这些关系之间的对应情况，详见标准与 UNC 类型比较的 64 位比较关系与并行类型比较的 64 位比较关系。

并行比较类型仅可用于一组受限制的关系与操作数。它们可以在两个寄存器或者一个寄存器与一个立即数之间进行相等与不等比较，或是在寄存器与 GR 0 之间进行不等比较。因为其中一个操作数为零时，它们没有大多作用，因此不提供无符号关系。对于并行不等式比较，硬件仅直接实现第一个操作数 (GR r2) 为 GR 0 情况。第二个操作数为 GR 0 的比较是伪操作，为此汇编器会切换寄存器指示符并使用相反的关系...
