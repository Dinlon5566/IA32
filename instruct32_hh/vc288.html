SGDT/SIDT - 存储全局/中断描述符表格寄存器

操作码

指令

说明

0F 01 /0

SGDT m

将 GDTR 存储到 m

0F 01 /1

SIDT m

将 IDTR 存储到 m

说明

将全局描述符表格寄存器 (GDTR) 或中断描述符表格寄存器 (IDTR) 中的内容存储到目标操作数。目标操作数是指定 6 字节内存位置。如果操作数大小属性为 32 位，则寄存器的 16 位限制字段存储到内存位置的 2 个低位字节，32 位基址存储到 4 个高位字节。如果操作数大小属性为 16 位，则限制字段存储在 2 个低位字节，24 位基址存储在第三、四及五字节，第六字节使用 0 填充。

SGDT 与 SIDT 指令仅在操作系统软件中有用；不过它们也可以在应用程序中使用，而不会导致生成异常。

如需有关加载 GDTR 与 IDTR 的详细信息，请参阅本章中的 LGDT/LIDT - 加载全局/中断描述符表格寄存器。

英特尔(R) 体系结构兼容性

如果不引用高 8 位，则 16 位形式的 SGDT 与 SIDT 指令与英特尔(R) 286 处理器兼容。英特尔(R) 286 处理器使用 1 填充这些位；奔腾(R) Pro、奔腾、Intel486™ 及 Intel386™ 处理器使用 0 填充这些位。

操作

IF instruction is IDTR
THEN
IF OperandSize 16
THEN
DEST[0:15] IDTR(Limit);
DEST[16:39] IDTR(Base); (* 24 bits of base address loaded; *)
DEST[40:47] 0;
ELSE (* 32-bit Operand Size *)
DEST[0:15] IDTR(Limit);
DEST[16:47] IDTR(Base); (* full 32-bit base address loaded *)
FI;
ELSE (* instruction is SGDT *)
IF OperandSize 16
THEN
DEST[0:15] GDTR(Limit);
DEST[16:39] GDTR(Base); (* 24 bits of base address loaded; *)
DEST[40:47] 0;
ELSE (* 32-bit Operand Size *)
DEST[0:15] GDTR(Limit);
DEST[16:47] GDTR(Base); (* full 32-bit base address loaded *)
FI; FI;

影响的标志

无。

保护模式异常

#UD - 如果目标操作数是寄存器。

#GP(0) - 如果目标位于不可写的段。如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。如果 DS、ES、FS、或 GS 寄存器用于访问内存，并且它包含空的段选择器。

#SS(0) - 如果内存操作数有效地址超出 SS 段限制。

#PF(错误代码) - 如果发生页错误。

#AC(0) - 如果在 CPL 为 3 且启用对齐检查的情况下发生未对齐的内存访问。

实地址模式异常

#UD - 如果目标操作数是寄存器。

#GP - 如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。

#SS - 如果内存操作数有效地址超出 SS 段限制。

虚 8086 模式异常

#UD - 如果目标操作数是寄存器。

#GP(0) - 如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。

#SS(0) - 如果内存操作数有效地址超出 SS 段限制。

#PF(错误代码) - 如果发生页错误。

#AC(0) - 如果在启用对齐检查的情况下发生未对齐的内存访问。
