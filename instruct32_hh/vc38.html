CMPPS - 比较压缩单精度浮点值

操作码

指令

说明

0F C2 /r ib

CMPPS xmm1, xmm2/m128, imm8

将 imm8 用作比较谓词，对 xmm2/mem 中的压缩单精度浮点值与 xmm1 中的压缩单精度浮点值执行比较。

说明

对源操作数（第二个操作数）与目标操作数（第一个操作数）中的四个压缩单精度浮点值执行 SIMD 比较，并将比较结果返回给目标操作数。比较谓词操作数（第三个操作数）指定要在每对压缩值上执行的比较类型。比较结果是四个全 1（比较结果为真）或全 0（比较结果为假）双字掩码。源操作数可以是 XMM 寄存器或 128 位内存位置。目标操作数是 XMM 寄存器。比较谓词操作数是一个 8 位立即数，其中头 3 位定义要执行的比较类型（请参阅 CMPPD 与 CMPPS 指令的比较谓词）；立即数的位 4 到 7 保留。

由于全 0 掩码对应浮点值 +0.0，全 1 掩码对应浮点值 QNaN，因此如果随后的计算型指令将目标操作数中的掩码结果用作输入操作数，并不会生成错误。

有些比较（如大于、大于或等于、不大于、不大于或等于关系）只能通过软件模拟来实现。对于这些比较，程序员必须交换操作数，在必要时通过复制寄存器来保护正在目标寄存器中的数据，然后使用不同的谓词执行比较。用于这些模拟的谓词在标题“模拟”下列出。

编译器与汇编器除实现三操作数 CMPPS 指令之外，还可能实现以下双操作数伪操作：

SIMD 浮点异常

操作数为 SNaN 时无效、非规格化。

保护模式异常

#GP(0) - CS、DS、ES、FS 或 GS 段中的内存操作数有效地址非法。如果内存操作数未对齐 16 字节边界，不论是哪一段。

#SS(0) - SS 段中的地址非法。

#PF(错误代码) - 页错误。

#NM - 如果 CR0 中的 TS 设置为 1。

#XM - 如果发生无掩码 SIMD 浮点异常，且 CR4 中的 OSXMMEXCPT 是 1。

#UD - 如果发生无掩码 SIMD 浮点异常，且 CR4 中的 OSXMMEXCPT 是 0。如果 CR0 中的 EM 设置为 1。如果 CR4 中的 OSFXSR 是 0。如果 CPUID 功能标志 SSE 为 0。

#GP(0) - 如果内存操作数未对齐 16 字节边界，不论是哪一段。中断 13 - 如果操作数的任何部分出现在从 0 到 0FFFFH 的有效地址空间之外。

#NM - 如果 CR0 中的 TS 设置为 1。

#XM - 如果发生无掩码 SIMD 浮点异常，且 CR4 中的 OSXMMEXCPT 是 1。

#UD - 如果发生无掩码 SIMD 浮点异常，且 CR4 中的 OSXMMEXCPT 是 0。如果 CR0 中的 EM 设置为 1。如果 CR4 中的 OSFXSR 是 0。如果 CPUID 功能标志 SSE 为 0。
