FPREM - 部分余数

操作码

指令

说明

D9 F8

FPREM

将 ST(0) 替换成 ST(0) 除以 ST(1) 的余数。

说明

计算将 ST(0) 寄存器中的值（被除数）除以 ST(1) 寄存器中的值（除数或模）所得到的余数，结果存储到 ST(0)。余数表示以下值：

Remainder ST(0) - (Q * ST(1))

这里，Q 是一个整数值，通过将 [ST(0) / ST(1)] 的实数商向零截断得到。余数的符号与被除数的相同。除非计算部分余数（如下所述），否则余数的值比模的小。

此指令会产生精确的结果；不会产生精度（不精确）异常，且不受取整控制的影响。下表显示计算各类数值的余数时得到的结果（假设未发生下溢）。

备注：F 表示有限实数。

* 表示浮点算术操作数无效 (#IA) 异常。

** 表示浮点除零 (#Z) 异常。

结果为 0 时，其符号与被除数的相同。模为 时，结果与 ST(0) 中的值相同。

FPREM 指令不计算 IEEE 754 标准中规定的余数。IEEE 规定的余数可以使用 FPREM1 指令计算。提供 FPREM 指令是为了同英特尔(R) 8087 及 Intel287 数学协处理器保持兼容。

FPREM 指令因计算余数的方式而得名“部分余数”。此指令通过反复执行减法得到余数。不过在指令的一次执行中，它可以将 ST(0) 的指数减去不超过 63 的值。如果指令成功产生小于模的余数，则操作完成，并清除 FPU 状态字中的 C2 标志。否则将设置 C2，此时 ST(0) 中的结果称为部分余数。部分余数的指数将比原始被除数的指数至少小 32。软件可以重复执行指令（将 ST(0) 中的部分余数用作被除数），直到清除 C2。（请注意，执行这样的余数计算循环时，在循环的指令之间，需要 FPU 且优先级更高的中断例程可能会强制进行上下文切换）。

FPREM 指令的重要用途之一是减少周期函数的参数。减法完成时，指令将商的三个最低有效位存储到 FPU 状态字的 C3、C1 及 C0 标志。由于减少正切函数的参数（使用模 /4）时，会将原始角度定位到单位圆的八个扇区之一，所以此信息非常重要。

操作

D exponent(ST(0)) - exponent(ST(1));
IF D < 64
THEN
Q Integer(TruncateTowardZero(ST(0) / ST(1)));
ST(0) ST(0) - (ST(1) * Q);
C2 0;
C0, C3, C1 LeastSignificantBits(Q); (* Q2, Q1, Q0 *)
ELSE
C2 1;
N an implementation-dependent number between 32 and 63;
QQ Integer(TruncateTowardZero((ST(0) / ST(1)) / 2(D-N)));
ST(0) ST(0) - (ST(1) * QQ * 2(D-N));
FI;

影响的 FPU 标志

C0 - 设置为商的位 2 (Q2)。

C1 - 如果发生堆栈下溢，则设置为 0；否则设置为商的最低有效位 (Q0)。

C2 - 如果减法完成，则设置为 0；如果未完成，则设置为 1。

C3 - 设置为商的位 1 (Q1)。

浮点异常

#IS - 发生堆栈下溢。

#IA - 源操作数是 SNaN 值、模是 0、被除数是 ，或是不支持的格式。

#D - 源操作数是非规格化值。

#U - 结果对于目标格式而言太小。

保护模式异常

#NM - 如果 CR0 中的 EM 或 TS 设置为 1。

实地址模式异常

#NM - 如果 CR0 中的 EM 或 TS 设置为 1。

虚 8086 模式异常

#NM - 如果 CR0 中的 EM 或 TS 设置为 1。
