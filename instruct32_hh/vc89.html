说明

比较寄存器 ST(0) 的内容与源操作数的值，并根据结果设置 FPU 状态字中的条件代码标志 C0、C2 及 C3（请参阅下表）。源操作数可以是数据寄存器或内存位置。如果没有给出源操作数，则比较 ST(0) 中的值与 ST(1) 中的值。忽略 0 的符号，因此 -0.0 +0.0。

备注：*如果生成无掩码算术操作数无效 (#IA) 异常，则不设置标志。

此指令会检查比较的数值的类别（请参阅本章中的 FXAM - 检查）。如果任何一个操作数是 NaN，或是不支持的格式，则触发算术操作数无效异常 (#IA)；如果屏蔽此异常，则条件标志设置为“无序”。如果未屏蔽算术操作数无效异常，则不设置条件代码标志。

执行比较操作之后，FCOMP 指令弹出寄存器堆栈；执行比较操作之后，FCOMPP 指令弹出寄存器堆栈两次。要弹出寄存器堆栈，处理器将 ST(0) 寄存器标记为空，并使堆栈指针 (TOP) 递增 1。

FCOM 指令执行的操作与 FUCOM 指令相同。唯一的区别在于处理 QNaN 操作数的方式。有一个操作数是或两个操作数都是 NaN，或是不支持的格式时，FCOM 指令触发算术操作数无效异常 (#IA)。FUCOM 指令执行的操作与 FCOM 指令相同，只是它不会针对 QNaN 生成算术操作数无效异常。

影响的 FPU 标志

C1 - 如果发生堆栈下溢，则设置为 1；否则清除为 0。C0、C2、C3 - 请参阅上表。

浮点异常

#IS - 发生堆栈下溢。

#IA - 一个操作数是，或两个操作数都是 NaN，或是采用不支持的格式。寄存器标记为空。

#D - 一个操作数是，或两个操作数都是非规格化值。

保护模式异常

#GP(0) - 如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。如果 DS、ES、FS 或 GS 寄存器包含空的段选择器。

#SS(0) - 如果内存操作数有效地址超出 SS 段限制。#NM - 如果 CR0 中的 EM 或 TS 设置为 1。

#PF(错误代码) - 如果发生页错误。

#AC(0) - 如果在当前特权级别为 3 且启用对齐检查的情况下进行未对齐的内存引用。

实地址模式异常

#GP - 如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。

#SS - 如果内存操作数有效地址超出 SS 段限制。

#NM - 如果 CR0 中的 EM 或 TS 设置为 1。

虚 8086 模式异常

#GP(0) - 如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。

#SS(0) - 如果内存操作数有效地址超出 SS 段限制。

#NM - 如果 CR0 中的 EM 或 TS 设置为 1。

#PF(错误代码) - 如果发生页错误。

#AC(0) - 如果在启用对齐检查的情况下进行未对齐的内存引用。
