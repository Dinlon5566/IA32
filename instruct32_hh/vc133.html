FYL2XP1

将 ST(1) 替换成 ST(1) * log 2 (ST(0) + 1.0)，并弹出寄存器堆栈

说明

以 epsilon 为底，计算 (ST(1) * log2 ST(0) + 1.0)) 的对数，结果存储到寄存器 ST(1)，并弹出 FPU 寄存器堆栈。ST(0) 中的源操作数必须在以下范围内：

ST(1) 中的源操作数范围是从 - 到 +。如果 ST(0) 操作数超出可接受范围，则结果未定义，软件不应依赖生成的异常。在某些情况下，ST(0) 超出范围时可能会生成异常，但此行为取决于具体的版本，具有一定的不确定性。

* 表示浮点操作无效 (#IA) 异常。

此指令能够为接近 0 的各种 epsilon 值 [寄存器 ST(0) 中的值] 提供最佳精度。对于很小的 epsilon () 值，同将 (+1) 用作 FYL2X 指令的参数相比，使用 FYL2XP1 指令可以保留更多的有效位。在复利与养老金计算中，经常可以看到 (+1) 表达式。通过在 ST(1) 源操作数中保存比例系数，很容易将结果转换成另一个底数的对数值。以下等式用于计算特定对数底数的比例系数，其中 n 是 FYL2XP1 指令的结果所需的对数底数：

影响的 FPU 标志

C1 - 如果发生堆栈下溢，则设置为 0。如果生成不精确的结果异常 (#P)，则表示取整方向：

0 非向上取整；1 向上取整。

C0、C2、C3 - 未定义。

浮点异常

#IS - 发生堆栈下溢。

#IA - 操作数是 SNaN 值或不支持的格式。

#D - 源操作数是非规格化值。

#U - 结果对于目标格式而言太小。

#O - 结果对于目标格式而言太大。

#P - 值无法按目标格式精确表示。

保护模式异常

#NM - 如果 CR0 中的 EM 或 TS 设置为 1。

实地址模式异常

#NM - 如果 CR0 中的 EM 或 TS 设置为 1。
