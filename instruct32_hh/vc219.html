OUT - 输出到端口

操作码

指令

说明

E6 ib

OUT imm8, AL

将 AL 中的字节输出到 I/O 端口地址 imm8

E7 ib

OUT imm8, AX

将 AX 中的字输出到 I/O 端口地址 imm8

E7 ib

OUT imm8, EAX

将 EAX 中的双字输出到 I/O 端口地址 imm8

EE

OUT DX, AL

将 AL 中的字节输出到 DX 中的 I/O 端口地址

EF

OUT DX, AX

将 AX 中的字输出到 DX 中的 I/O 端口地址

EF

OUT DX, EAX

将 EAX 中的双字输出到 DX 中的 I/O 端口地址

说明

将值从第二个操作数（源操作数）复制到目标操作数（第一个操作数）指定的 I/O 端口。源操作数可以是 AL、AX 或 EAX 寄存器，具体取决于访问端口的大小（分别为 8 位、16 位或 32 位）；目标操作数可以是字节立即数或 DX 寄存器。使用字节立即数时，可以访问 0 到 255 之间的 I/O 端口地址；将 DX 寄存器用作源操作数时，可以访问 0 到 65,535 之间的 I/O 端口。

对于 8 位 I/O 端口，访问的端口大小由操作码确定；对于 16 位或 32 位 I/O 端口，则由指令的操作数大小属性确定。

在机器代码级别上，访问 8 位 I/O 端口时，I/O 指令更短。这里，端口地址的高 8 位将是 0。

此指令只能用于访问处理器 I/O 地址空间中的 I/O 端口。如需有关访问 I/O 地址空间中的 I/O 端口的详细信息，请参阅“IA-32 英特尔(R) 体系结构软件开发人员手册”第 1 卷第 9 章“输入/输出”。

英特尔(R) 体系结构兼容性

执行 OUT 指令之后，奔腾® 处理器会在开始执行下一条指令之前，确保 EWBE# 引脚已通过采样激活。（请注意，如果 EWBE# 处于未忽动状态，则可以预取此指令，但是要到通过采用激活 EWBE# 引脚之后，才可以执行它）。只有奔腾处理器系列有 EWBE# 引脚；其它 IA-32 处理器都没有。

操作

IF ((PE 1) AND ((CPL > IOPL) OR (VM 1)))
THEN (* Protected mode with CPL > IOPL or virtual-8086 mode *)
IF (Any I/O Permission Bit for I/O port being accessed 1)
THEN (* I/O operation is not allowed *)
#GP(0);
ELSE ( * I/O operation is allowed *)
DEST SRC; (* Writes to selected I/O port *)
FI;
ELSE (Real Mode or Protected Mode with CPL IOPL *)
DEST SRC; (* Writes to selected I/O port *)
FI;

影响的标志

无。

保护模式异常

#GP(0) - 如果 CPL 大于（特权较低）I/O 特权级别 (IOPL)，并且访问的 I/O 端口的 TSS 中相应的任何 I/O 权限位是 1。

实地址模式异常

无。

虚 8086 模式异常

#GP(0) - 如果访问的 I/O 端口的 TSS 中的任何 I/O 权限位是 1。
