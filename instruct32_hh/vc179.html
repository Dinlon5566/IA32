MOV - 移入/移出调试寄存器

操作码

指令

说明

0F 21/r

MOV r32,DR0-DR7

将调试寄存器移到 r32

0F 23 /r

MOV DR0-DR7,r32

将 r32 移到调试寄存器

说明

将调试寄存器（DR0、DR1、DR2、DR3、DR4、DR5、DR6 或 DR7）的内容移到通用寄存器，或者相反。不论操作数大小属性如何，这些指令的操作数大小总是 32 位。（如需有关调试寄存器的标志与字段的详细说明，请参阅“IA-32 英特尔(R) 体系结构软件开发人员手册”第 3 卷第 14 章“调试与性能监视”）。

这些指令必须在特权级别为 0 时执行，或是在实地址模式中执行。

清除寄存器 CR4 中的调试扩展 (DE) 标志时，这些指令按照同 Intel386™ 与 Intel486™ 处理器兼容的方式操作调试寄存器。在这个模式中，引用 DR4 与 DR5 将分别引用 DR6 与 DR7。CR4 中的 DE 设置为 1 时，试图引用 DR4 与 DR5 会导致操作码未定义 (#UD) 异常。（CR4 寄存器是从奔腾(R) 处理器开始新增到“英特尔(R) 体系结构”中的）。

在操作码级别，ModR/M 字节中的 reg 字段指定加载或读取的调试寄存器。mod 字段中的两个位总是 11。r/m 字段指定加载或读取的通用寄存器。

操作

IF ((DE 1) and (SRC or DEST DR4 or DR5))
THEN
#UD;
ELSE
DEST SRC;

影响的标志

OF、SF、ZF、AF、PF 及 CF 标志未定义。

保护模式异常

#GP(0) - 如果当前特权级别不是 0。

#UD - 如果 CR4 中的 DE（调试扩展）位设置为 1，且执行涉及 DR4 或 DR5 的 MOV 指令。

#DB - 如果在调试寄存器 DR7 中的 GD 标志设置为 1 时访问任何调试寄存器。

实地址模式异常

#UD - 如果 CR4 中的 DE（调试扩展）位设置为 1，且执行涉及 DR4 或 DR5 的 MOV 指令。

#DB - 如果在调试寄存器 DR7 中的 GD 标志设置为 1 时访问任何调试寄存器。

虚 8086 模式异常

#GP(0) - 调试寄存器在虚 8086 模式中无法加载或读取。
