说明

将内存操作数指定的字节、字或双字与 AL、AX 或 EAX 寄存器中的值，并根据结果设置 EFLAGS 寄存器中的状态标志。内存操作数地址从 DS:EDI 或 DS:DI 寄存器中读取（具体取决于指令的地址大小属性，分别为 32 或 16）。ES 段不能使用跨段前缀覆盖。

在汇编代码级别，此指令允许采用两种形式：“显式操作数”形式与“无操作数”形式。显式操作数形式（使用 SCAS 助记符指定）可以明确指定内存操作数。这里，内存操作数应该是表示操作数值大小与位置的符号。接着，自动选择与源操作数大小匹配的寄存器操作数（对于字节比较，选择 AL 寄存器；对于字比较，选择 AX；对于双字比较，选择 EAX）。提供此显式操作数形式是为了便于记录；不过请注意，此形式提供的记录可能会令人误解。也就是说，内存操作数符号必须指定操作数（字节、字或双字）的正确类型（大小），但不必指定正确的位置。位置总是由 ES:(E)DI 寄存器指定，在执行比较字符串指令之前，必须先正确加载这些寄存器。

无操作数形式提供“简短形式”的字节、字及双字版 SCAS 指令。这里，同样假设 ES:(E)DI 是内存操作数，假设 AL、AX 或 EAX 寄存器是寄存器作数。两个操作数的大小使用助记符选择：SCASB（字节比较）、SCASW（字比较）或 SCASD（双字比较）。

比较之后，根据 EFLAGS 寄存器中 DF 标志的设置，(E)DI 寄存器会自动递增或递减。（如果 DF 标志是 0，则 (E)DI 寄存器递增；如果 DF 标志是 1，则 (E)DI 寄存器递减）。对于字节操作，(E)DI 寄存器递增或递减 1；对于字操作，递增或递减 2；对于双字操作，递增或递减 4。

通过 REP 前缀，SCAS、SCASB、SCASW 及 SCASD 指令可用于整块比较 ECX 字节、字或双字。不过，更常见的情况是在 LOOP 结构中使用这些指令，此结构会在进行下一个比较之前，根据状态标志的设置执行一些操作。如需有关 REP 前缀的说明，请参阅本章的 REP/REPE/REPZ/REPNE/REPNZ - 重复字符串操作前缀。

操作

IF (byte cmparison)
THEN
temp AL - SRC;
SetStatusFlags(temp);
THEN IF DF 0
THEN (E)DI (E)DI + 1;
ELSE (E)DI (E)DI - 1;
FI;
ELSE IF (word comparison)
THEN
temp AX - SRC;
SetStatusFlags(temp)
THEN IF DF 0
THEN (E)DI (E)DI + 2;
ELSE (E)DI (E)DI - 2;
FI;
ELSE (* doubleword comparison *)
temp EAX - SRC;
SetStatusFlags(temp)
THEN IF DF 0
THEN (E)DI (E)DI + 4;
ELSE (E)DI (E)DI - 4;
FI;
FI;
FI;

影响的标志

OF、SF、ZF、AF PF 及 CF 标志根据比较的临时结果设置。

保护模式异常

#GP(0) - 如果内存操作数有效地址超出 ES 段限制。如果 ES 寄存器包含空的段选择器。如果 ES 段中给出的内存操作数有效地址非法。

#PF(错误代码) - 如果发生页错误。
