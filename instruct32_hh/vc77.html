DIVSD - 标量双精度浮点除法

操作码

指令

说明

F2 0F 5E /r

DIVSD xmm1, xmm2/m64

将 xmm1 中的低位双精度浮点值除以 xmm2/mem64中的低位双精度浮点值。

说明

将目标操作数（第一个操作数）中的低位双精度浮点值除以源操作数（第二个操作数）中的低位双精度浮点值，双精度浮点结果存储到目标操作数。源操作数可以是 XMM 寄存器或 64 位内存位置。目标操作数是 XMM 寄存器。目标操作数的高位四字保持不变。如需有关标量双精度浮点操作的示意图，请参阅“IA-32 英特尔(R) 体系结构软件开发人员手册”第 1 卷的“图 11-4”。

操作

DEST[63-0] DEST[63-0] / SRC[63-0];
* DEST[127-64] remains unchanged *;

SIMD 浮点异常

上溢、下溢、无效、除零、精度、非规格化。

保护模式异常

#GP(0) - CS、DS、ES、FS 或 GS 段中的内存操作数有效地址非法。

#SS(0) - SS 段中的地址非法。

#PF(错误代码) - 页错误。

#NM - 如果 CR0 中的 TS 设置为 1。

#XM - 如果发生无掩码 SIMD 浮点异常，且 CR4 中的 OSXMMEXCPT 是 1。

#UD - 如果发生无掩码 SIMD 浮点异常，且 CR4 中的 OSXMMEXCPT 是 0。如果 CR0 中的 EM 设置为 1。如果 CR4 中的 OSFXSR 是 0。如果 CPUID 功能标志 SSE2 是 0。

#AC - 如果在当前特权级别为 3 时进行未对齐的内存引用。

实地址模式异常

中断 13 - 如果操作数的任何部分出现在从 0 到 0FFFFH 的有效地址空间之外。

#NM - 如果 CR0 中的 TS 设置为 1。

#XM - 如果发生无掩码 SIMD 浮点异常，且 CR4 中的 OSXMMEXCPT 是 1。

#UD - 如果发生无掩码 SIMD 浮点异常，且 CR4 中的 OSXMMEXCPT 是 0。如果 CR0 中的 EM 设置为 1。如果 CR4 中的 OSFXSR 是 0。如果 CPUID 功能标志 SSE2 是 0。

虚 8086 模式异常

与“实地址模式”中的异常相同。

#PF(错误代码) - 页错误。

#AC - 如果在当前特权级别为 3 时进行未对齐的内存引用。
