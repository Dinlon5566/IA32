LTR - 加载任务寄存器

操作码

指令

说明

0F 00 /3

LTR r/m16

将 r/m16 加载到任务寄存器

说明

将源操作数加载到任务寄存器的段选择器字段。源操作数（通用寄存器或内存位置）包含指向任务状态段 (TSS) 的段选择器。段选择器加载到任务寄存器之后，处理器使用段选择器在全局描述符表格 (GDT) 中确定 TSS 的段描述符的位置。接着，它将 TSS 的段限制与基址从段描述符加载到任务寄存器。任务寄存器指向的任务标记为忙，但不发生任务切换。

提供的 LTR 指令用于操作系统软件；不应该在应用程序中使用它。它只能在 CPL 为 0 时在保护模式中执行。它常用在初始化代码，建立要执行的第一个任务。

操作数大小属性不影响此指令。

操作

IF SRC[Offset) > descriptor table limit OR IF SRC[type) global
THEN #GP(segment selector);
FI;
Read segment descriptor;
IF segment descriptor is not for an available TSS THEN #GP(segment selector); FI;
IF segment descriptor is not present THEN #NP(segment selector);
TSSsegmentDescriptor(busy) 1;
(* Locked read-modify-write operation on the entire descriptor when setting busy flag *)
TaskRegister(SegmentSelector) SRC;
TaskRegister(SegmentDescriptor) TSSSegmentDescriptor;

影响的标志

无。

保护模式异常

#GP(0) - 如果当前特权级别不是 0。如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。如果 DS、ES、FS、或 GS 寄存器用于访问内存，并且它包含空的段选择器。

#GP(选择器) - 如果源选择器指向的段不是 TSS 段，或是指向已处于忙状态的任务对应的段。如果选择器指向 LDT，或是超出 GDT 的限制。

#NP(选择器) - 如果 TSS 标记为不存在。

#SS(0) - 如果内存操作数有效地址超出 SS 段限制。

#PF(错误代码) - 如果发生页错误。

实地址模式异常

#UD - LTR 指令在实地址模式中无法识别。

虚 8086 模式异常

#UD - LTR 指令在虚 8086 模式中无法识别。
