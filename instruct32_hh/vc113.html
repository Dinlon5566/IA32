FSAVE/FNSAVE - 存储 x87 FPU 状态

操作码

指令

说明

9B DD /6

FSAVE m94/108byte

在检查未决的无掩码浮点异常之后，将 FPU 状态存储到 m94byte 或 m108byte。然后重新初始化 FPU。

DD /6

FNSAVE* m94/108byte

将 FPU 环境存储到 m94byte 或 m108byte，不检查未决的无掩码浮点异常。然后重新初始化 FPU。

说明

将当前的 FPU 状态（操作环境与寄存器堆栈）存储到内存中指定的目标，然后重新初始化 FPU。在存储 FPU 状态之前，FSAVE 指令检查并处理未决的无掩码浮点异常，FNSAVE 指令则不会。

FPU 操作环境由 FPU 控制字、状态字、标记字、指令指针、数据指针以及上一个操作码组成。“IA-32 英特尔(R) 体系结构软件开发人员手册”第 1 卷的“图 7-13”到“图 7-16”显示存储的环境在内存中的布局，具体取决于处理器的操作模式（保护模式或实模式）以及当前的操作数大小属性（16 位或 32 位）。在虚 8086 模式中，使用实模式布局。FPU 寄存器堆栈的内容存储在紧跟操作环境映像之后的 80 个字节中。

保存的映像反映的是，FPU 在指令流中 FSAVE/FNSAVE 指令前面的所有浮点指令都执行之后的状态。

已经保存 FPU 状态之后，FPU 复位成使用 FINIT/FNINIT 指令设置的相同默认值（请参阅本章中的 FINIT/FNINIT - 初始化浮点单元）。

FSAVE/FNSAVE 指令通常用在以下场合：操作系统需要执行上下文切换，异常处理需要使用 FPU，或是应用程序需要向过程传递“干净的”FPU。

英特尔体系结构兼容性

对于英特尔(R) 奔腾(R) 处理器之前的英特尔数学协处理器与 FPU，在试图读取前面的 FSAVE/FNSAVE 指令存储的内存映像之前，应该先执行 FWAIT 指令。这个 FWAIT 指令有助于确保存储操作已经完成。

在 MS-DOS* 操作系统兼容模式中操作奔腾或 Intel486™ 处理器时，FNSAVE 指令（在特殊的情况下）可能会在执行（以便处理未决的 FPU 异常）之前中断。如需有关这些情况的说明，请参阅“IA-32 英特尔体系结构软件开发人员手册”第 1 卷附录 D 中标题为“无等待 FPU 指令可以在窗口中发生 FPU 中断”的部分。在奔腾 Pro 处理器上，FNSAVE 指令不能以此方式中断。

操作

(* Save FPU State and Registers *)
DEST[FPUControlWord) FPUControlWord;
DEST[FPUStatusWord) FPUStatusWord;
DEST[FPUTagWord) FPUTagWord;
DEST[FPUDataPointer) FPUDataPointer;
DEST[FPUInstructionPointer) FPUInstructionPointer;
DEST[FPULastInstructionOpcode) FPULastInstructionOpcode;
DEST[ST(0)) ST(0);
DEST[ST(1)) ST(1);
DEST[ST(2)) ST(2);
DEST[ST(3)) ST(3);
DEST[ST(4)) ST(4);
DEST[ST(5)) ST(5);
DEST[ST(6)) ST(6);
DEST[ST(7)) ST(7);
(* Initialize FPU *)
FPUControlWord 037FH;
FPUStatusWord 0;
FPUTagWord FFFFH;
FPUDataPointer 0;
FPUInstructionPointer 0;
FPULastInstructionOpcode 0;

影响的 FPU 标志

保存 C0、C1、C2 及 C3 标志，然后清除。

浮点异常

无。

保护模式异常

#GP(0) - 如果目标位于不可写的段。如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。如果 DS、ES、FS、或 GS 寄存器用于访问内存，并且它包含空的段选择器。

#SS(0) - 如果内存操作数有效地址超出 SS 段限制。

#NM - 如果 CR0 中的 EM 或 TS 设置为 1。

#PF(错误代码) - 如果发生页错误。

#AC(0) - 如果启用对齐检查并在当前特权级别为 3 时进行未对齐的内存引用。

实地址模式异常

#GP - 如果内存操作数的有效地址超出 CS、DS、ES、FS 或 GS 段限制。

#SS - 如果内存操作数有效地址超出 SS 段限制。

#NM - 如果 CR0 中的 EM 或 TS 设置为 1。

虚 8086 模式异常

#GP(0) - 如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。

#SS(0) - 如果内存操作数有效地址超出 SS 段限制。

#NM - 如果 CR0 中的 EM 或 TS 设置为 1。

#PF(错误代码) - 如果发生页错误。

#AC(0) - 如果在启用对齐检查的情况下进行未对齐的内存引用。
