说明

将第二个操作数（源操作数）复制到第一个操作数（目标操作数）。源操作数可以是立即数、通用寄存器、段寄存器或内存位置；目标寄存器可以是通用寄存器、段寄存器或内存位置。两个操作数的大小必须一样，可以是字节、字或双字。

MOV 指令不能用于加载 CS 寄存器。试图这样做会导致操作码无效异常 (#UD)。要加载 CS 寄存器，请使用远 JMP、CALL 或 RET 指令。

如果目标操作数是段寄存器 (DS、ES、FS、GS 或 SS)，则源操作数必须是有效的段选择器。在保护模式中，将段选择器移到段寄存器会导致将该段选择器关联的段描述符信息自动加载到段寄存器的隐藏（阴影）部分。加载此信息时，段选择器与段描述符信息生效（请参阅下面的“操作”算法）。段描述符数据从指定的段选择器对应的 GDT 或 LDT 项中获取。

空的段选择器（值为 0000 到 0003）可以加载到 DS、ES、FS 及 GS 寄存器，而不会导致保护异常。不过，随后试图引用段寄存器中加载空值的段时，都将导致一般保护性异常 (#GP)，并且不发生任何内存引用。

使用 MOV 指令加载 SS 寄存器时，会禁止所有中断，直到执行下一条指令之后。此操作可用于在发生中断之前，将堆栈指针与下一条指令加载到 ESP (MOV ESP, 堆栈指针值)。LSS 指令是更有效的加载 SS 与 ESP 寄存器的方法。

在 32 位模式中操作并在段寄存器与通用寄存器之间移动数据时，“英特尔(R) 体系结构”32 位处理器不要求配合 16 位操作数大小前缀（值为 66H 的字节）使用此指令，但如果使用标准形式的指令（如 MOV DS, AX），大多数汇编器都会插入此前缀。处理器将正确执行此指令，但通常要额外花费一个时钟周期。对于大多数汇编器而言，通过使用指令形式 MOV DS, EAX 可以避免插入这个不必要的 66H 前缀。处理器执行使用 32 位通用寄存器的此指令时，它会假定通用寄存器的 16 个最低有效位是目标操作数或源操作数。如果寄存器是目标操作数，则寄存器的两个高位字节中产生的值取决于具体实现的版本。对于奔腾(R) Pro 处理器，这两个高位字节用 0 填充；对于早期的 32 位“英特尔® 体系结构”处理器，这两个高位字节未定义。

DEST SRC;

在保护模式中，加载段寄存器会导致执行特殊的检查与操作，具体如下所述。这些检查的对象是段选择器及其指向的段描述符。

IF SS is loaded;
THEN
IF segment selector is null
THEN #GP(0);
FI;
IF segment selector index is outside descriptor table limits
OR segment selector's RPL CPL
OR segment is not a writable data segment
OR DPL CPL
THEN #GP(selector);
FI;
IF segment not marked present
THEN #SS(selector);
ELSE
SS segment selector;
SS segment descriptor;
FI;
FI;
IF DS, ES, FS, or GS is loaded with non-null selector;
THEN
IF segment selector index is outside descriptor table limits
OR segment is not a data or readable code segment
OR ((segment is a data or nonconforming code segment)
AND (both RPL and CPL > DPL))
THEN #GP(selector);
IF segment not marked present
THEN #NP(selector);
ELSE
SegmentRegister segment selector;
SegmentRegister segment descriptor;
FI;
FI;
IF DS, ES, FS, or GS is loaded with a null selector;
THEN
SegmentRegister segment selector;
SegmentRegister segment descriptor;
FI;

保护模式异常

#GP(0) - 如果试图将空的段选择器加载到 SS 寄存器。如果目标操作数位于不可写的段。如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。如果 DS、ES、FS 或 GS 寄存器包含空的段选择器。

#GP(选择器) - 如果段选择器索引超出描述符表格限制。如果加载 SS 寄存器，并且段选择器的 RPL 与段描述符的 DPL 不等于 CPL。如果加载 SS 寄存器，且指向的段不是可写的数据段。如果加载 DS、ES、FS 或 GS 寄存器，且指向的段不是数据段或可读代码段。如果加载 DS、ES、FS 或 GS 寄存器，且指向的段是数据段或非相容代码段，但 RPL 与 CPL 都大于 DPL。

#SS(0) - 如果内存操作数有效地址超出 SS 段限制。

#SS(选择器) - 如果加载 SS 寄存器且指向的段标记为不存在。

#NP - 如果加载 DS、ES、FS 或 GS 寄存器且指向的段标记为不存在。

#PF(错误代码) - 如果发生页错误。

#AC(0) - 如果启用对齐检查并在当前特权级别为 3 时进行未对齐的内存引用。

#UD - 如果试图加载 CS 寄存器。

#SS - 如果内存操作数有效地址超出 SS 段限制。

#UD - 如果试图加载 CS 寄存器。
