SYSEXIT - 从系统调用入口快速转出

操作码

指令

说明

0F 35

SYSEXIT

从“系统调用入口”转出

说明

SYSEXIT 指令是在奔腾(R) II 处理器上引入的“快速系统调用”功能的一部分。SYSEXIT 指令经过优化，能以最佳性能从保护环 0 (CPL 0) 转到保护环 3 (CPL 3)。

根据操作系统在特定的型号专用寄存器或通用寄存器中指定的值，SYSEXIT 指令设置以下寄存器。

CS 寄存器设置为 16 与 SYSENTER_CS_MSR 的值之和EIP 寄存器设置为 EDX 寄存器中包含的值。SS 寄存器设置为 24 与 SYSENTER_CS_MSR 的值之和ESP 寄存器设置为 ECX 寄存器中包含的值。

处理器不保存内核堆栈或返回地址信息，也不保存任何寄存器。

SYSENTER 与 SYSEXIT 指令不构成一对调用/返回；因此，用户代码执行的系统调用“存根”例程（通常在共享库或 DLL 中）必须负责还原所需的寄存器状态，以创建一对系统调用/返回。

SYSEXIT 指令总是转换到 CPL 3 普通保护模式用户级别。SYSEXIT 只能在 CPL 0 时从保护模式激活。此指令要求操作系统满足以下条件：

    目标环 3 代码段的 CS 选择器是 32 位，映射成普通 0-4 GB 地址空间，并且拥有执行、读取以及非相容权限。目标环 3 堆栈段的 SS 选择器是 32 位，映射成普通 0-4 GB 地址空间，并且拥有扩展、读取以及写入权限。

操作系统必须进行以下设置：

 

名称

说明

CS 选择器

目标环 3 CS 选择器。它被赋予 16 与 SYSENTER_CS_MSR 的值之和。

SS 选择器

目标环 3 SS 选择器。它被赋予 24 与 SYSENTER_CS_MSR 的值之和。

EIP

目标环 3 返回 EIP。这是目标入口，被赋予 EDX 寄存器中包含的值。

ESP

目标环 3 返回 ESP。这是目标入口，被赋予 ECX 寄存器中包含的值。

此项功能的存在与否由 CPUID 的位 11“SYSENTER 存在”(SEP) 指出。检测到存在 SEP 位的操作系统还必须判断处理器的系列与型号是否符合，以确保确实提供 SYSENTER/SYSEXIT 指令，具体如关于 SYSENTER 指令的说明所述。奔腾 Pro 处理器（型号 1）返回一组 SEP CPUID 功能位，但不支持 SYSENTER/SYSEXIT 指令。

操作

SYSEXIT
IF SYSENTER_CS_MSR == 0 THEN #GP(0)
IF CR0.PE == 0 THEN #GP(0)
IF CPL <> 0 THEN #GP(0)

// Changing CS:EIP and SS:ESP is required

CS.SEL := (SYSENTER_CS_MSR + 16) // Selector for return CS
CS.SEL.RPL := 3

// Set rest of CS to a fixed value
CS.SEL.BASE := 0 // Flat segment
CS.SEL.LIMIT := 0xFFFF // 4G limit
CS.SEL.G := 1 // 4 KB granularity
CS.SEL.S := 1
CS.SEL.TYPE_xCRA := 1011 // Execute, Read, Non-Conforming Code
CS.SEL.D := 1 // 32 bit code
CS.SEL.DPL := 3
CS.SEL.P := 1

SS.SEL := (SYSENTER_CS_MSR + 24)
SS.SEL.RPL := 3

// Set rest of SS to a fixed value
SS.SEL.BASE := 0 // Flat segment
SS.SEL.LIMIT := 0xFFFF // 4G limit
SS.SEL.G := 1 // 4 KB granularity
SS.SEL.S := 1
SS.SEL.TYPE_xCRA := 0011 // Expand Up, Read/Write, Data
SS.SEL.D := 1 // 32 bit stack
SS.SEL.DPL := 3
SS.SEL.CPL := 3
SS.SEL.P := 1

ESP := ECX
EIP := EDX

异常

#GP(0) - 如果 SYSENTER_CS_MSR 包含零。

数值异常

无。

保护模式异常

#GP(0) - 如果 CPL 不为零。

实地址模式异常

#GP(0) - 如果未启用保护模式。
