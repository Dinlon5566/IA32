说明

复位时 MXCSR 的默认值是 0x1f80。

位 5-0 表示是否检测到“数据流单指令多数据扩展指令集”异常。它们是“粘性”标志，可以通过使用 LDMXCSR 指令向这些字段写入零加以清除。如果 LDMXCSR 指令清除掩码位并设置相应的异常标志位，则不会立即生成异常。仅当执行下一条会导致此类异常的“数据流单指令多数据扩展指令集”指令时，才会发生异常。“数据流单指令多数据扩展指令集”对于每个异常仅使用一个异常标志。对于压缩数据类型中的异常，不提供单独的报告。在相同的指令中发生多个完全相同的异常情况下，将更新关联的异常标志，以指出至少发生其中一种情况。复位时这些标志都将清除。

位 12-7 配置数值异常掩码。如果设置异常类型对应的位，则屏蔽该异常类型；如果该位清除，则取消掩码。复位时将设置这些启用位，表示屏蔽所有数值异常。

位 14-13 给取整控制编码，提供普通就近模式的取整、定向取整以及截尾取整。取整控制影响算术指令与某些转换指令。RC 的编码如下所示：

复位时取整控制设置为就近取整。

位 15 (FZ) 用于打开“清除为零”模式（位设置为 1）。发生下溢时，打开“清除为零”模式时具有以下效果：

  • 返回结果零与正确结果的符号

  • 设置精度与下溢异常标志

IEEE 规定的下溢掩码响应是传递非规格化结果（即逐渐下溢）；因此“清除为零”模式与 IEEE 754 标准不兼容。提供它主要是出于性能方面的考虑。在经常发生下溢的应用程序中，稍微损失一点精度可以换来更快的执行速度。取消下溢异常掩码的优先级高于“清除为零”模式。这种安排表示，不论是否启用“清除为零”，在取消下溢异常的掩码时，处理器都会为生成下溢情况的“数据流单指令多数据扩展指令集”指令激活异常处理程序。

MXCSR 的其它位（位 31-16 与位 6）定义为保留位并处于清除状态；试图使用 FXRSTOR 或 LDMXCSR 指令向这些位写入非零值，将导致一般保护性异常。

线性地址对应于引用的内存数据的最低有效字节。

MXCSR m32;

异常

一般保护性错误 - 如果将非零值加载到保留位。

数值异常

无。

保护模式异常

#GP(0) - CS、DS、ES、FS 或 GS 段中的内存操作数有效地址非法。

#SS(0) - SS 段中的地址非法。

#PF(错误代码) - 页错误。

#UD - 如果 CR0.EM 1。

#NM - 如果 CR0 中的 TS 位设置为 1。

#AC - 未对齐的内存引用。要启用 #AC 异常，必须满足三个条件（CR0.AM 设置为 1；EFLAGS.AC 设置为 1；当前的 CPL 是 3）。#UD - 如果 CR4.OSFXSR（位 9） 0。 #UD - 如果 CPUID.XMM（EDX 位 25） 0。

实地址模式异常

中断 13 - 如果操作数的任何部分出现在从 0 到 0FFFFH 的有效地址空间之外。

#UD - 如果 CR0.EM 1。

#NM - 如果 CR0 中的 TS 位设置为 1。

#UD - 如果 CR4.OSFXSR（位 9） 0。 #UD 如果 CPUID.XMM（EDX 位 25） 0。

虚 8086 模式异常

与“实地址模式”中的异常相同。

#PF(错误代码) - 页错误。

#AC - 未对齐的内存引用。

注释

配合 Repeat 前缀 (F3H) 使用 LDMXCSR 属于保留用法。不同的处理器版本处理此前缀的方式可能会有所不同。配合此前缀使用 LDMXCSR 可能会同将来的处理器不兼容。
