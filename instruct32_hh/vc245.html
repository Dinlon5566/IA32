说明

将栈顶的值加载到目标操作数指定的位置，然后递增堆栈指针。目标操作数可以是通用寄存器、内存位置或段寄存器。

堆栈段的地址大小属性确定堆栈指针大小（16 位或 32 位 - 源地址大小），当前代码段的操作数大小属性确定堆栈指针的递增量（2 字节或 4 字节）。例如，如果这些地址大小属性与操作数大小属性为 32，则 32 位 ESP 寄存器（堆栈指针）递增 4，如果这些属性为 16，则 16 位 SP 寄存器递增 2。（堆栈段的段描述符中的 B 标志确定堆栈的地址大小属性，当前代码段的段描述符中的 D 标志与前缀确定目标操作数的操作数大小属性与地址大小属性）。

如果目标操作数是段寄存器 DS、ES、FS、GS 或 SS 之一，则加载到寄存器的值必须是有效的段选择器。在保护模式中，将段选择器弹入段寄存器时，会导致将该段选择器关联的段描述符信息自动加载到段寄存器的隐藏（阴影）部分，并且会使选择器与描述符信息生效（请参阅下面的“操作”部分）。

空值 (0000-0003) 可以弹入 DS、ES、FS 或 GS 寄存器，且不会导致一般保护性错误。不过，随后试图对段寄存器中加载了空值的段进行任何引用时，都将导致一般保护性异常 (#GP)。在这种情况下，不会发生任何内存引用，并且段寄存器保存的值为空。

POP 指令无法将值弹入 CS 寄存器。要将堆栈中的值加载到 CS 寄存器，请使用 RET 指令。

如果将 ESP 寄存器用作基址寄存器，以便在内存中确定目标操作数的地址，则 POP 指令会在递增 ESP 寄存器之后，计算操作数的有效地址。对于 16 位堆栈的情况，如果执行 POP 指令之后，ESP 通过舍位变成 0h，则产生的内存写入位置取决于具体的处理器系列。

POP ESP 指令会在旧栈顶的数据写入目标操作数之前，递增堆栈指针 (ESP)。

POP SS 指令会抑制所有中断（包括 NMI 中断），直到执行下一条指令之后。此操作可以确保依次执行 POP SS 与 MOV ESP, EBP 指令，而不会在中断期间遇到失效的堆栈。不过，使用 LSS 指令才是加载 SS 与 ESP 寄存器的首选方法。

操作

IF StackAddrSize 32
THEN
IF OperandSize 32
THEN
DEST SS:ESP; (* copy a doubleword *)
ESP ESP + 4;
ELSE (* OperandSize 16*)
DEST SS:ESP; (* copy a word *)
ESP ESP + 2;
FI;
ELSE (* StackAddrSize 16* )
IF OperandSize 16
THEN
DEST SS:SP; (* copy a word *)
SP SP + 2;
ELSE (* OperandSize 32 *)
DEST SS:SP; (* copy a doubleword *)
SP SP + 4;
FI;
FI;

在保护模式中，加载段寄存器会导致执行特殊的检查与操作，具体如下所述。这些检查的对象是段选择器及其指向的段描述符。

IF SS is loaded;
THEN
IF segment selector is null
THEN #GP(0);
FI;
IF segment selector index is outside descriptor table limits
OR segment selector's RPL CPL
OR segment is not a writable data segment
OR DPL CPL
THEN #GP(selector);
FI;
IF segment not marked present
THEN #SS(selector);
ELSE
SS segment selector;
SS segment descriptor;
FI;
FI;
IF DS, ES, FS, or GS is loaded with non-null selector;
THEN
IF segment selector index is outside descriptor table limits
OR segment is not a data or readable code segment
OR ((segment is a data or nonconforming code segment)
AND (both RPL and CPL > DPL))
THEN #GP(selector);
IF segment not marked present
THEN #NP(selector);
ELSE
SegmentRegister segment selector;
SegmentRegister segment descriptor;
FI;
FI;
IF DS, ES, FS, or GS is loaded with a null selector;
THEN
SegmentRegister segment selector;
SegmentRegister segment descriptor;
FI;

影响的标志

无。

保护模式异常

#GP(0) - 如果试图将空的段选择器加载到 SS 寄存器。如果目标操作数位于不可写的段。如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。如果 DS、ES、FS、或 GS 寄存器用于访问内存，并且它包含空的段选择器。

#GP(选择器) - 如果段选择器索引超出描述符表格限制。如果加载 SS 寄存器，并且段选择器的 RPL 与段描述符的 DPL 不等于 CPL。如果加载 SS 寄存器，且指向的段不是可写的数据段。如果加载 DS、ES、FS 或 GS 寄存器，且指向的段不是数据段或可读代码段。如果加载 DS、ES、FS 或 GS 寄存器，且指向的段是数据段或非相容代码段，但 RPL 与 CPL 都大于 DPL。

#SS(0) - 如果当前栈顶不在堆栈段内。如果内存操作数有效地址超出 SS 段限制。

#SS(选择器) - 如果加载 SS 寄存器且指向的段标记为不存在。

#NP - 如果加载 DS、ES、FS 或 GS 寄存器且指向的段标记为不存在。

#PF(错误代码) - 如果发生页错误。

#AC(0) - 如果在当前特权级别为 3 且启用对齐检查的情况下进行未对齐的内存引用。

实地址模式异常

#GP - 如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。

虚 8086 模式异常

#GP(0) - 如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。

#PF(错误代码) - 如果发生页错误。

#AC(0) - 如果在启用对齐检查的情况下进行未对齐的内存引用。
