IN - 从端口输入

操作码

指令

说明

E4 ib

IN AL,imm8

从 imm8 I/O 端口地址将字节输入 AL

E5 ib

IN AX,imm8

从 imm8 I/O 端口地址将字节输入 AX

E5 ib

IN EAX,imm8

从 imm8 I/O 端口地址将字节输入 EAX

EC

IN AL,DX

从 DX 中的 I/O 端口将字节输入 AL

ED

IN AX,DX

从 DX 中的 I/O 端口将字输入到 AX

ED

IN EAX,DX

从 DX 中的 I/O 端口将双字输入到 EAX

说明

从第二个操作数（源操作数）指定的 I/O 端口将值复制到目标操作数（第一个操作数）。源操作数可以是字节立即数或 DX 寄存器，目标操作数可以是寄存器 AL、AX 或 EAX，具体取决于访问的端口大小（分别为 8、16 或 32 位）。将 DX 寄存器用作源操作数时，可以访问从 0 到 65,535 的 I/O 端口地址；使用字节立即数时，可以访问从 0 到 255 的 I/O 端口地址。

访问 8 位 I/O 端口时，操作码确定端口大小；访问 16 与 32 位 I/O 端口时，操作数大小确定端口大小。

在机器代码级别上，访问 8 位 I/O 端口时，I/O 指令更短。这里，端口地址的高 8 位将是 0。

此指令只能用于访问处理器 I/O 地址空间中的 I/O 端口。如需有关访问 I/O 地址空间中的 I/O 端口的详细信息，请参阅“IA-32 英特尔(R) 体系结构软件开发人员手册”第 1 卷第 9 章“输入/输出”。

操作

IF ((PE 1) AND ((CPL > IOPL) OR (VM 1)))
THEN (* Protected mode with CPL > IOPL or virtual-8086 mode *)
IF (Any I/O Permission Bit for I/O port being accessed 1)
THEN (* I/O operation is not allowed *)
#GP(0);
ELSE ( * I/O operation is allowed *)
DEST SRC; (* Reads from selected I/O port *)
FI;
ELSE (Real Mode or Protected Mode with CPL IOPL *)
DEST SRC; (* Reads from selected I/O port *)
FI;

影响的标志

无。

保护模式异常

#GP(0) - 如果 CPL 大于（特权较低）I/O 特权级别 (IOPL)，并且访问的 I/O 端口的 TSS 中相应的任何 I/O 权限位是 1。

实地址模式异常

无。

虚 8086 模式异常

#GP(0) - 如果访问的 I/O 端口的 TSS 中的任何 I/O 权限位是 1。
