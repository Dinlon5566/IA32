说明

比较寄存器 ST(0) 中的值与整型源操作数，根据结果设置 FPU 状态字中的条件代码标志 C0、C2 及 C3（请参阅下表）。执行比较之前，整数值会转换成扩展实数格式。

这些指令执行“无序比较”。无序比较还检查比较的数值的类别（请参阅 FXAM - 检查）。如果其中一个操作数是 NaN，或是未定义的格式，则条件标志设置为“无序”。

忽略 0 的符号，因此 -0.0 +0.0。

比较之后，FICOMP 指令弹出寄存器堆栈。要弹出寄存器堆栈，处理器将 ST(0) 寄存器标记为空，并将堆栈指针 (TOP) 递增 1。

影响的 FPU 标志

C1 - 如果发生堆栈下溢，则设置为 0；否则设置为 0。

C0、C2、C3 - 请参阅上表。

浮点异常

#IS - 发生堆栈下溢。

#IA - 一个操作数是，或两个操作数都是 NaN，或是采用不支持的格式。

#D - 一个操作数是，或两个操作数都是非规格化值。

保护模式异常

#GP(0) - 如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。如果 DS、ES、FS 或 GS 寄存器包含空的段选择器。

#SS(0) - 如果内存操作数有效地址超出 SS 段限制。

#NM - 如果 CR0 中的 EM 或 TS 设置为 1。

#PF(错误代码) - 如果发生页错误。

#AC(0) - 如果在当前特权级别为 3 且启用对齐检查的情况下进行未对齐的内存引用。

实地址模式异常

#GP - 如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。

#SS - 如果内存操作数有效地址超出 SS 段限制。

#NM - 如果 CR0 中的 EM 或 TS 设置为 1。

虚 8086 模式异常

#GP(0) - 如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。

#SS(0) - 如果内存操作数有效地址超出 SS 段限制。

#NM - 如果 CR0 中的 EM 或 TS 设置为 1。

#PF(错误代码) - 如果发生页错误。

#AC(0) - 如果在启用对齐检查的情况下进行未对齐的内存引用。
