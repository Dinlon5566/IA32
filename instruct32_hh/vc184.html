MOVDQU - 移動未對齊的雙四字

操作碼

指令

說明

F3 0F 6F /r

MOVDQU xmm1, xmm2/m128

將未對齊的雙四字從 xmm2/m128 移到 xmm1。

F3 0F 7F /r

MOVDQU xmm2/m128, xmm1

將未對齊的雙四字從 xmm1 移到 xmm2/m128。

說明

將雙四字從源運算元（第二個運算元）移到目標運算元（第一個運算元）。此指令可以用於在 XMM 暫存器與 128 位記憶體位置之間移入/移出雙四字，或是在兩個 XMM 暫存器之間移動。源運算元或目標運算元是記憶體運算元時，運算元可以不對齊 16 位元組邊界，並且不會導致產生一般保護性異常 (#GP)。

對於已知與對齊 16 位元組邊界的記憶體位置，要在其中移入/移出雙四字，請使用 MOVDQA 指令。

操作

DEST SRC;
* #GP if SRC or DEST unaligned memory operand *;

英特爾(R) C++ 編譯器等價內部函式

MOVDQU void _mm_storeu_si128 ( __m128i *p, __m128i a) MOVDQU __m128i _mm_loadu_si128 ( __m128i *p)

SIMD 浮點異常

無。

保護模式異常

#GP(0) - 如果記憶體運算元有效地址超出 CS、DS、ES、FS 或 GS 段限制。

#SS(0) - 如果記憶體運算元有效地址超出 SS 段限制。

#NM - 如果 CR0 中的 TS 設定為 1。

#UD - 如果 CR0 中的 EM 設定為 1。如果 CR4 中的 OSFXSR 是 0。如果 CPUID 功能標誌 SSE2 是 0。

#PF(錯誤程式碼) - 如果發生頁錯誤。

實地址模式異常

#GP(0) - 如果運算元的任何部分出現在 0 到 FFFFH 的有效地址空間之外。

#NM - 如果 CR0 中的 TS 設定為 1。

#UD - 如果 CR0 中的 EM 設定為 1。如果 CR4 中的 OSFXSR 是 0。如果 CPUID 功能標誌 SSE2 是 0。

虛 8086 模式異常

與“實地址模式”中的異常相同。

#PF(錯誤程式碼) - 頁錯誤。
