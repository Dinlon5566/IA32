6D

INSW

从 DX 指定的 I/O 端口将字输入 ES:(E)DI 指定的内存位置

说明

从源操作数（第二个操作数）指定的 I/O 端口将数据复制到目标操作数（第一个操作数）。源操作数是从 DX 寄存器读取的 I/O 端口地址（从 0 到 65,535）。目标操作数是内存位置，其地址从 ES:EDI 或 ES:DI 寄存器读取（具体取决于指令的地址大小属性，分别是 32 或 16）。（ES 段不能使用跨段前缀覆盖）。对于 8 位 I/O 端口，访问的 I/O 端口的大小（即源操作数与目标操作数的大小）由操作码确定；对于 16 或 32 位 I/O 端口，则由指令的操作数大小属性确定。

在汇编代码级别，此指令允许采用两种形式：“显式操作数”形式与“无操作数”形式。显式操作数形式（使用 INS 助记符指定）可以明确指定源操作数与目标操作数。这里，源操作数必须是 "DX"，目标操作数应该是表示 I/O 端口与目标地址大小的符号。提供此显式操作数形式是为了便于记录；不过请注意，此形式提供的记录可能会令人误解。也就是说，目标操作数符号必须指定操作数（字节、字或双字）的正确类型（大小），但不必指定正确的位置。位置总是由 ES:(E)DI 寄存器指定，在执行 INS 指令之前，必须先正确加载这些寄存器。

无操作数形式提供“简短形式”的字节、字及双字版 INS 指令。这里，处理器假设 DX 是源操作数，ES:(E)DI 是目标操作数。I/O 端口的大小可以选择使用助记符指定：INSB（字节）、INSW（字）或 INSD（双字）。

从 I/O 端口将字节、字或双字传输到内存位置之后，根据 EFLAGS 寄存器中的 DF 标志的设置，(E)DI 寄存器自动递增或递减。（如果 DF 标志是 0，则 (E)DI 寄存器递增；如果 DF 标志是 1，则 (E)DI 寄存器递减）。对于字节操作，(E)DI 寄存器递增或递减 1；对于字操作，递增或递减 2；对于双字操作，递增或递减 4。

通过加上 REP 前缀，可以使用 INS、INSB、INSW 及 INSD 指令整块输入 ECX 字节、字或双字。如需有关 REP 前缀的说明，请参阅本章的 REP/REPE/REPZ/REPNE/REPNZ - 重复字符串操作前缀。

这些指令仅用于访问处理器 I/O 地址空间中的 I/O 端口。如需有关访问 I/O 地址空间中的 I/O 端口的详细信息，请参阅“IA-32 英特尔(R) 体系结构软件开发人员手册”第 1 卷第 9 章“输入/输出”。

操作

IF ((PE 1) AND ((CPL > IOPL) OR (VM 1)))
THEN (* Protected mode with CPL > IOPL or virtual-8086 mode *)
IF (Any I/O Permission Bit for I/O port being accessed 1)
THEN (* I/O operation is not allowed *)
#GP(0);
ELSE ( * I/O operation is allowed *)
DEST SRC; (* Reads from I/O port *)
FI;
ELSE (Real Mode or Protected Mode with CPL IOPL *)
DEST SRC; (* Reads from I/O port *)
FI;
IF (byte transfer)
THEN IF DF 0
THEN (E)DI (E)DI + 1;
ELSE (E)DI (E)DI - 1;
FI;
ELSE IF (word transfer)
THEN IF DF 0
THEN (E)DI (E)DI + 2;
ELSE (E)DI (E)DI - 2;
FI;
ELSE (* doubleword transfer *)
THEN IF DF 0
THEN (E)DI (E)DI + 4;
ELSE (E)DI (E)DI - 4;
FI;
FI;
FI;

影响的标志

无。

保护模式异常

#GP(0) - 如果 CPL 大于（特权较低）I/O 特权级别 (IOPL)，并且访问的 I/O 端口的 TSS 中相应的任何 I/O 权限位是 1。如果目标位于不可写的段。如果给定的 ES 段中的内存操作数有效地址非法。

#PF(错误代码) - 如果发生页错误。

#AC(0) - 如果启用对齐检查并在当前特权级别为 3 时进行未对齐的内存引用。

实地址模式异常

#GP - 如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。

#SS - 如果内存操作数有效地址超出 SS 段限制。

虚 8086 模式异常

#GP(0) - 如果访问的 I/O 端口的 TSS 中的任何 I/O 权限位是 1。

#PF(错误代码) - 如果发生页错误。

#AC(0) - 如果在启用对齐检查的情况下进行未对齐的内存引用。
