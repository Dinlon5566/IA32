内部 API

使用 MMX™ 技术内部函数以及“数据流单指令多数据扩展指令集”与“第二代数据流单指令多数据扩展指令集”内部函数进行编码的好处是，可以使用 C 函数调用与 C 变量的语法替代硬件寄存器。这使得您不再需要管理寄存器与编写汇编程序。而且编译器还优化指令调度，从而使可执行文件运行速度更快。对于新指令集中的每个计算型与数据操作指令，都有一个对应的 C 内部函数直接去实现它。内部函数可供指定算法的底层实现（指令选择），而将指令调度与寄存器分配交给编译器去完成。

MMX™ 技术内部函数

MMX 技术内部函数基于新的 __m64 数据类型，以代表 MMX 技术寄存器的特定内容。您可以按照字节、短整数、32 位值或 64 位对象的形式指定值。不过，__m64 数据类型并非 ANSI C 基本数据类型，因此必须遵循以下使用限制：

  • __m64 数据只能在赋值语句的左侧作为返回值或参数使用。它不能同其它算术表达式（"+"、">>" 等）一起使用。

  • 在聚集中使用 __m64 对象，例如联合（用于访问字节元素）与结构；可以取 __m64 对象的地址。

  • 只能在本指南与 Intel(R) C++ Compiler User's Guide With Support for the Streaming SIMD Extensions 2 【英特尔(R) C++ 编译器（含第二代数据流单指令多数据扩展指令集支持）用户指南，订购编号 718195-2001】介绍的 MMX 技术内部函数中使用 __m64 数据。

数据流单指令多数据扩展指令集与第二代数据流单指令多数据扩展指令集内部函数

所有的“数据流单指令多数据扩展指令集”与“第二代数据流单指令多数据扩展指令集”内部函数都会利用到奔腾(R) III 与奔腾 4 处理器的 xmm 寄存器。这些内部函数支持三种数据类型：__m128、__m128d 及 __m128i。

  • __m128 数据类型用于代表“数据流单指令多数据扩展指令集”内部函数所用“数据流单指令多数据扩展指令集”寄存器的内容。这既可以是四个压缩单精度浮点值，也可以是一个标量单精度数字。

  • __m128d 数据类型存放两个 64 位浮点值（双精度）。

  • __m128i 数据类型可以存放十六个 8 位、八个 16 位、四个 32 位或两个 64 位整数值。

编译器将 __m128、__m128d 及 __m128 局部与全局数据对齐堆栈的 16 字节边界。要对齐整数、浮点或双精度数组，可以使用 declspec 语句，具体介绍详见“英特尔(R) C++ 编译器（含第二代数据流单指令多数据扩展指令集支持）用户指南”（订购编号 718195-2001）。

__m128 数据类型并非 ANSI C 基本数据类型，因此它的使用有以下限制：

__m128、__m128d 及 __m128i 只能在赋值语句的左侧作为返回值或参数使用。不要在其它算术表达式（如 "+" 与 ">>"）中使用它。不要使用常值初始化 __m128、__m128d 及 __m128i；没有办法表示 128 位常数。在聚集中使用 __m128、__m128d 及 __m128i 对象，如联合（例如，要访问浮点元素）与结构。可以取这些对象的地址。仅在本用户指南介绍的内部函数中使用 __m128、__m128d 及 __m128i 数据。

编译器将 __m128、__m128d 及 __m128i 局部数据对齐堆栈的 16 字节边界。全局 __m128 数据也在 16 字节边界上对齐。（要对齐浮点数组，可以使用下节介绍的对齐 declspec）。不管使用压缩还是标量数据，新指令集都按照相同的方式对待 SIMD 浮点寄存器，因此没有 __m32 数据类型可以代表您可能需要的标量数据。对于标量操作，应该使用 __m128 对象与内部函数的“标量”形式；编译器与处理器使用 32 位内存引用实现这些操作。

后缀 ps 与 ss 用于代表“压缩单精度”与“标量单精度”操作。压缩浮点值按从右到左的顺序表示，最低位的字（最右侧）用于标量操作：[z, y, x, w]。要理解内存存储如何反映这点，请参阅下例。

操作

float a[4] { 1.0, 2.0, 3.0, 4.0 };
__m128 t _mm_load_ps(a);

产生的结果跟以下语句的相同：

__m128 t _mm_set_ps(4.0, 3.0, 2.0, 1.0);

也就是说，

t [ 4.0, 3.0, 2.0, 1.0 ]

其中“标量”元素是 1.0。

有些内部函数因为需要多条指令来实现，因此可称作是“复合的”。使用内部函数编写程序时，应该去熟悉“数据流单指令多数据扩展指令集”、“第二代数据流单指令多数据扩展指令集”以及 MMX 技术提供的硬件功能。

记住以下三个要点：

特定的内部函数（如 _mm_loadr_ps 与 _mm_cmpgt_ss）不受指令集直接支持。尽管这些内部函数是非常方便的编程辅助手段，但还是要注意它们的实现成本。作为 __m128 对象加载或存储的数据通常是按照 16 字节对齐。由于指令的性质，有些内部函数要求它们的参数必须是立即数，也就是说是整型常数（常值）。作用于两个 NaN（非数值）参数的算术运算的结果未定义。因此，使用 NaN 参数的浮点操作产生的行为可能与对应的汇编指令的预期行为有所不同。
