FPATAN - 部分反正切

操作码

指令

说明

D9 F3

FPATAN

将 ST(1) 替换成 arctan(ST(1)/ST(0))，并弹出寄存器堆栈

说明

将寄存器 ST(1) 中的源操作数除以寄存器 ST(0) 中的源操作数，然后计算商的反正切，结果存储到 ST(1)，并弹出 FPU 寄存器堆栈。寄存器 ST(0) 中的结果的符号与源操作数 ST(1) 的相同，并且数值比 + 小。

FPATAN 指令返回 X 轴同原点及点 (X,Y) 确定的直线之间的角度，其中 Y（纵坐标）是 ST(1)，X（横坐标）是 ST(0)。此角度同 X 与 Y 的符号都有关，而不只是同比率 Y/X 的符号有关。这是因为点 (-X,Y) 位于第二象限，得到的角度在 /2 与 之间，而点 (X,-Y) 位于第四象限，得到的角度在 0 与 -/2 之间。点 (-X,-Y) 位于第三象限，得到的角度在 -/2 与 - 之间。

备注：F 表示有限实数。

*“IA-32 英特尔(R) 体系结构软件开发人员手册”第 1 卷“表 7-20”指定比率 0/0 与 / 会生成浮点算术操作无效异常，并且，如果设置此异常的掩码，则返回无穷大的实数值。使用 FPATAN 指令时，实际上不使用除法计算 0/0 或 / 值。相反，这两个变量的反正切根据通用的标准数学公式推导，此公式能够将复数用作参数。在这个复变量公式中，反正切 (0,0) 等都有非常明确的值。在以只能接受实数参数的 FPU 函数为基础，来开发计算带复数参数的超越函数库时，需要用到这些值。

FPATAN 可以接受的源操作数范围没有任何限制。

英特尔(R) 体系结构兼容性

在 80287 数学协处理器中，此指令的源操作数范围限制如下：

0 |ST(1)| < |ST(0)| < +

操作

ST(1) arctan(ST(1) / ST(0));
PopRegisterStack;

影响的 FPU 标志

C1 - 如果发生堆栈下溢，则设置为 0。如果生成不精确的结果异常 (#P)，则表示取整方向：0 非向上取整；1 向上取整。

C0、C2、C3 - 未定义。

浮点异常

#IS - 发生堆栈下溢。

#IA - 源操作数是 SNaN 值或不支持的格式。

#D - 源操作数是非规格化值。

#U - 结果对于目标格式而言太小。

#P - 值无法按目标格式精确表示。

保护模式异常

#NM - 如果 CR0 中的 EM 或 TS 设置为 1。

实地址模式异常

#NM - 如果 CR0 中的 EM 或 TS 设置为 1。

虚 8086 模式异常

#NM - 如果 CR0 中的 EM 或 TS 设置为 1。
