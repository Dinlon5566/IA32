CVTTSS2SI - 標量單精度浮點值轉換成雙字整數，帶截斷

操作碼

指令

說明

F3 0F 2C /r

CVTTSS2SI r32, xmm/m32

運用截斷處理將 xmm/m32 中的一個單精度浮點值轉換成 r32 中的一個有符號雙字整數。

說明

將源運算元（第二個運算元）中的單精度浮點值轉換成目標運算元（第一個運算元）中的有符號雙字整數。源運算元可以是 XMM 暫存器或 32 位記憶體位置。目標運算元是通用暫存器。源運算元是 XMM 暫存器時，單精度浮點值包含在暫存器的低位雙字中。

轉換不精確時，返回截斷結果。如果轉換的結果大於最大有符號雙字整數，則觸發浮點無效異常；如果設定此異常的掩碼，則返回無窮大整數值 (80000000H)。

操作

DEST[31-0] Convert_Single_Precision_Floating_Point_To_Integer_Truncate(SRC[31-0]);

英特爾(R) C++ 編譯器等價內部函式

int_mm_cvttss_si32(__m128d a)

SIMD 浮點異常

無效、精度。

保護模式異常

#GP(0) - CS、DS、ES、FS 或 GS 段中的記憶體運算元有效地址非法。

#SS(0) - SS 段中的地址非法。

#PF(錯誤程式碼) - 頁錯誤。

#NM - 如果 CR0 中的 TS 設定為 1。

#XM - 如果發生無掩碼 SIMD 浮點異常，且 CR4 中的 OSXMMEXCPT 是 1。

#UD - 如果發生無掩碼 SIMD 浮點異常，且 CR4 中的 OSXMMEXCPT 是 0。如果 CR0 中的 EM 設定為 1。如果 CR4 中的 OSFXSR 是 0。如果 CPUID 功能標誌 SSE 為 0。

#AC - 如果在目前特權級別為 3 時進行未對齊的記憶體引用。

實地址模式異常

中斷 13 - 如果運算元的任何部分出現在從 0 到 0FFFFH 的有效地址空間之外。

#NM - 如果 CR0 中的 TS 設定為 1。

#XM - 如果發生無掩碼 SIMD 浮點異常，且 CR4 中的 OSXMMEXCPT 是 1。

#UD - 如果發生無掩碼 SIMD 浮點異常，且 CR4 中的 OSXMMEXCPT 是 0。如果 CR0 中的 EM 設定為 1。如果 CR4 中的 OSFXSR 是 0。如果 CPUID 功能標誌 SSE 為 0。

虛 8086 模式異常

與“實地址模式”中的異常相同。

#PF(錯誤程式碼) - 頁錯誤。

#AC - 如果在目前特權級別為 3 時進行未對齊的記憶體引用。
