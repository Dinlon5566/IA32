指令

说明

FXSAVE m512byte

将 x87 FPU、MMX™ 技术、“数据流单指令多数据扩展指令集”以及“第二代数据流单指令多数据扩展指令集”状态保存到 m512byte。

说明

FXSAVE 指令将 x87 FPU、MMX™ 技术、“数据流单指令多数据扩展指令集”、“第二代数据流单指令多数据扩展指令集”的数据、控制及状态寄存器的当前状态写入目标操作数。目标是 512 字节内存位置。

此指令不检查未决的无掩码浮点异常（与 FNSAVE 的操作类似）。与 FSAVE/FNSAVE 指令不同，保存状态之后，处理器保留 x87 FPU、MMX 技术、数据流单指令多数据扩展指令集以及数据流单指令多数据扩展指令集状态的内容。保存数据结构（请参阅 FXSAVE 与 FXRSTOR 内存区域布局）使用 little-endian（低位在先、高位在后）字节顺序，与内存中的安排一样，字节偏移量如每行最右侧的列所述。

浮点保存区域中有三个字段包含保留位，这些位没有在表中指出：

FOP：低 11 位包含操作码，高 5 位保留。IP 与 DP 32 位模式：32 位 IP 偏移量。16 位模式：低 16 位是 IP 偏移量，高 16 位保留。

FXSAVE 指令通常用在以下场合：操作系统需要执行上下文切换，异常处理需要使用浮点、MMX 技术以及“数据流单指令多数据扩展指令集”单元。由于它会保留当前状态，因此应用程序不能使用它向过程传递“干净的”FP 状态。要提供此功能，应用程序必须在 FXSAVE 之后明确执行 FINIT 指令。

除 FTW 之外，所有的 x87 FPU 字段都保留与 FSAVE 相同的内部格式。

与 FSAVE 不同，FXSAVE 只保存 FTW 有效位，而不是整个 x87-FP FTW 字段。FTW 位以非 TOS 相对顺序保存，这表示总是先保存 FR0，然后是 FR1、FR2，依次类推。例如，如果 TOS=4，且只有 ST0、ST1 与 ST2 有效，则 FSAVE 会按以下格式保存 FTW 字段：

ST3 ST2 ST1 ST0 ST7 ST6 ST5 ST4 (TOS=4) FR7 FR6 FR5 FR4 FR3 FR2 FR1 FR0 11 xx xx xx 11 11 11 11

其中 xx 是 (00, 01, 10) 中的一个。(11) 表示空栈元素，00、01 及 10 分别表示“有效值”、“零”及“特殊值”。

在本例中，FXSAVE 会保存以下矢量：

FR7 FRits6 FR5 FR4 FR3 FR2 FR1 FR0 0 1 1 1 0 0 0 0

J 位定义为有效位中小数点位置左侧的一位二进制整数。M 位定义为有效位中小数部分的最高有效位（即正好在小数点右侧的位）。

M 位是有效位中小数部分的最高有效位时，如果小数部分全零，则 M 位必须是 0。

如果 FXSAVE 指令的前一条指令是不使用内存操作数的浮点指令，则 FXSAVE 指令不会将 DP 字段写入/更新到 FXSAVE 映像。

MXCSR 保存 SIMD 浮点“控制/状态寄存器”的内容。如需有关此字段的详细说明，请参阅 LDMXCSR 指令。

字段 XMM0-XMM7 包含寄存器 XMM0-XMM7 的内容，格式与在寄存器中完全相同。

如果未设置 CR4.OSFXSR 位，则处理器不将保存映像中的“数据流单指令多数据扩展指令集”字段（XMM0-XMM7 与 MXCSR）加以保存。这个 CR4 位必须设置，才能执行“数据流单指令多数据扩展指令集”。

目标 m512byte 假设与 16 字节边界对齐。如果 m512byte 未与 16 字节边界对齐，FXSAVE 会生成一般保护性异常。

注释

使用 FXSAVE 保存并使用 FRSTOR 还原状态，以及使用 FSAVE 保存并使用 FXRSTOR 还原状态，都将导致处理器的状态无法正确还原。地址大小前缀对地址计算的影响跟通常情况一样，但它不会影响 FXSAVE 映像的格式。

配合 Repeat(F2H, F3H) 与“操作数大小”(66H) 前缀使用 FXSAVE 属于保留用法。不同的处理器版本处理这些前缀的方式可能会有所不同。配合这些前缀使用 FXSAVE 可能会同将来的处理器不兼容。
