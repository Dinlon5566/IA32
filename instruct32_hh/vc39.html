说明

比较第一个源操作数指定的字节、字或双字与第二个源操作数指定的字节、字或双字，并根据结果设置 EFLAGS 寄存器中的状态标志。两个源操作数都位于内存。第一个源操作数的地址从 DS:ESI 或 DS:SI 寄存器中读取（具体取决于指令的地址大小属性，分别为 32 或 16）。第二个源操作数的地址从 ES:EDI 或 ES:DI 寄存器中读取（具体也取决于指令的地址大小属性）。DS 段可以用跨段前缀覆盖，但 ES 段不能覆盖。

在汇编代码级别，此指令允许采用两种形式：“显式操作数”形式与“无操作数”形式。显式操作数形式（由 CMPS 助记符指定）可以明确指定两个源操作数。这里，源操作数应该是表示源操作数值大小与位置的符号。提供此显式操作数形式是为了便于记录；不过请注意，此形式提供的记录可能会令人误解。也就是说，源操作数符号必须指定操作数（字节、字或双字）的正确类型（大小），但不必指定正确的位置。源操作数的位置总是由 DS:(E)SI 与 ES:(E)DI 寄存器指定，而且必须在执行字符串比较指令之前正确加载这些寄存器。

无操作数形式提供“简短形式”的字节、字及双字版 CMPS 指令。这里，处理器还假定 DS:(E)SI 与 ES:(E)DI 寄存器指定源操作数的位置。源操作数的大小使用助记符来选择：CMPSB（字节比较）、CMPSW（字比较）或 CMPSD（双字比较）。

比较之后，根据 EFLAGS 寄存器中 DF 标志的设置，(E)SI 与 (E)DI 寄存器会自动递增或递减。（如果 DF 标志为 0，则 (E)SI 与 (E)DI 寄存器递增；如果 DF 标志为 1，则 (E)SI 与 (E)DI 寄存器递减）。对于字节操作，寄存器递增或递减 1；对于字操作，递增或递减 2；对于双字操作，则递增或递减 4。

通过以 REP 前缀打头，CMPS、CMPSB、CMPSW 及 CMPSD 指令可用于成块比较 ECX 字节、字或双字。不过，更常见的情况是在 LOOP 结构中使用这些指令，此结构会在进行下一个比较之前，根据状态标志的设置执行一些操作。如需有关 REP 前缀的说明，请参阅本章的 REP/REPE/REPZ/REPNE/REPNZ - 重复字符串操作前缀。

影响的标志

CF、OF、SF、ZF、AF 及 PF 标志根据比较的临时结果设置。

保护模式异常

#GP(0) - 如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。如果 DS、ES、FS 或 GS 寄存器包含空的段选择器。

#SS(0) - 如果内存操作数有效地址超出 SS 段限制。

#PF(错误代码) - 如果发生页错误。

#AC(0) - 如果启用对齐检查并在当前特权级别为 3 时进行未对齐的内存引用。

实地址模式异常

#GP - 如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。

#SS - 如果内存操作数有效地址超出 SS 段限制。

虚 8086 模式异常

#GP(0) - 如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。

#SS(0) - 如果内存操作数有效地址超出 SS 段限制。

#PF(错误代码) - 如果发生页错误。

#AC(0) - 如果在启用对齐检查的情况下进行未对齐的内存引用。
