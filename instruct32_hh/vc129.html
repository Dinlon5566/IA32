FXRSTOR m512byte

从 m512byte 加载 x87 FPU、MMX™ 技术、“数据流单指令多数据扩展指令集”以及“第二代数据流单指令多数据扩展指令集”状态。

说明

从源操作数重新加载 x87 FPU、MMX™ 技术、“数据流单指令多数据扩展指令集”、“第二代数据流单指令多数据扩展指令集”的数据、控制及状态寄存器。源操作数是 512 字节内存位置。此数据应该先使用 FXSAVE 指令写入内存。FXSAVE 与 FXRSTOR 内存区域布局显示状态信息在内存中的布局。浮点保存区域中有三个字段包含保留位，这些位没有在表中指出：

FOP：低 11 位包含操作码，高 5 位保留。IP 与 DP 32 位模式：32 位 IP 偏移量。16 位模式：低 16 位是 IP 偏移量，高 16 位保留。

如果 MXCSR 状态包含无掩码异常并且设置了相应的状态标志，则加载它时不会导致声言浮点错误条件。只有下一次发生此无掩码异常时，才会导致声言错误条件。

MXCSR 的一些位（位 31-16 与位 6 ）定义为保留位或清除位；试图向这些位写入非零值将导致一般保护性异常。

FXRSTOR 指令不清除未决的 x87 FPU 异常，这与 FRSTOR 指令不同。要在加载新的操作环境时检查并触发异常，请先后使用 FXRSTOR 指令与 FWAIT 指令。

如果未设置 CR4.OSFXSR 位，则不将保存映像中的“数据流单指令多数据扩展指令集”字段（XMM0-XMM7 与 MXCSR）加载到处理器。这个 CR4 位必须设置，才能执行“数据流数据流单指令多数据扩展指令集”。

使用 FXSAVE 保存并使用 FRSTOR 还原状态，以及使用 FSAVE 保存并使用 FXRSTOR 还原状态，都将导致处理器的状态无法正确还原。地址大小前缀对地址计算的影响跟通常的情况一样，但它不影响 FXRSTOR 映像的格式。

保护模式异常

#GP(0) - CS、DS、ES、FS 或 GS 段中的内存操作数有效地址非法，或是试图将非零值加载到 MXCSR 字段中的保留位。如果内存操作数未对齐 16 字节边界，不论是哪一段。

#SS(0) - SS 段中的地址非法。

#PF(错误代码) - 页错误。

#NM - 如果 CR0.EM 1。

#NM - 如果 CR0 中的 TS 位设置为 1。

#UD - 如果指令带 LOCK 覆盖前缀

#AC - 如果在当前特权级别为 3 时进行未对齐的内存引用。如果启用 #AC（且 CPL 为 3），则发出 #AC 信号所表示的含义是不确定的，随具体版本而异。在不发出 #AC 信号的所有版本中，将生成一般保护性错误，而不是发出信号。此外启用 #AC 时，对齐检查的宽度也随具体版本而异；例如，对于给定的版本，在发生 2 字节未对齐时，可能会发出 #AC 信号，而发生其它未对齐（4、8 或 16 字节）情况时，则可能会发出 #GP 信号。
