说明

将目标操作数与源操作数相乘，乘积存储到目标位置。目标操作数总是 FPU 数据寄存器；源操作数可以是 FPU 数据寄存器或内存位置。内存中的源操作数可以是单精度实数、双精度实数、字整数或短整数格式。

无操作数版的此指令将 ST(1) 寄存器的内容乘以 ST(0) 寄存器的内容，乘积存储到 ST(1) 寄存器。单操作数版将 ST(0) 寄存器的内容乘以内存位置的内容（实数或整数值），乘积存储到 ST(0) 寄存器。双操作数版将 ST(0) 寄存器的内容乘以 ST(i) 寄存器的内容（反之亦然），乘积存储到第一个操作数（目标操作数）指定的寄存器。

存储乘积之后，FMULP 指令执行额外的操作，即弹出 FPU 寄存器堆栈。为了弹出寄存器堆栈，处理器将 ST(0) 寄存器标记为空，并使堆栈指针 (TOP) 递增 1。无操作数版的浮点乘法指令总是弹出寄存器堆栈。在某些汇编器中，此指令的助记符是 FMUL 而不是 FMULP。

执行乘法之前，FIMUL 指令将整型源操作数转换成扩展的实数格式。

即使有一个或多个乘数的值为 0 或 ，结果的符号也总是源操作数符号的“异或”。源操作数为整数 0 时，它被当作 +0。

影响的 FPU 标志

C1 - 如果发生堆栈下溢，则设置为 0。如果生成不精确的结果异常 (#P) 错误，则表示取整方向：

0 非向上取整；1 向上取整。

C0、C2、C3 - 未定义。

浮点异常

#IS - 发生堆栈下溢。

#IA - 操作数是 SNaN 值或不支持的格式。一个操作数是 ±0，另一个是 ±。

#D - 源操作数是非规格化值。

#U - 结果对于目标格式而言太小。

#O - 结果对于目标格式而言太大。

#P - 值无法按目标格式精确表示。

保护模式异常

#GP(0) - 如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。如果 DS、ES、FS、或 GS 寄存器用于访问内存，并且它包含空的段选择器。

#SS(0) - 如果内存操作数有效地址超出 SS 段限制。

#NM - 如果 CR0 中的 EM 或 TS 设置为 1。

#PF(错误代码) - 如果发生页错误。

#AC(0) - 如果启用对齐检查并在当前特权级别为 3 时进行未对齐的内存引用。

实地址模式异常

#GP - 如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。

#SS - 如果内存操作数有效地址超出 SS 段限制。

#NM - 如果 CR0 中的 EM 或 TS 设置为 1。

虚 8086 模式异常

#GP(0) - 如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。

#SS(0) - 如果内存操作数有效地址超出 SS 段限制。

#NM - 如果 CR0 中的 EM 或 TS 设置为 1。

#PF(错误代码) - 如果发生页错误。

#AC(0) - 如果在启用对齐检查的情况下进行未对齐的内存引用。
