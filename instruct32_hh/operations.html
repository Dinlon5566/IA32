操作

这部分包含指令的算法说明（使用伪代码编写）。伪代码使用类似 Algol 或 Pascal 语言的表示法。算法由以下元素组成：

注释括在成对的符号 "(*" 与 "*)" 中。复合语句括在关键字中，如 if 语句的 IF、THEN、ELSE 及 FI；do 语句的 DO 与 OD；或是 case 语句的 CASE ...OF 及 ESAC。寄存器名称暗示寄存器的内容。方括号内的寄存器名称暗示该寄存器包含其地址的位置的内容。例如，ES:[DI] 表示 ES 段相对地址在寄存器 DI 中的位置的内容。[SI] 表示寄存器 SI 包含的相对于 SI 的默认段 (DS) 或覆盖段的地址的内容。通用寄存器名称中 "E" 两侧的圆括号（如 (E)SI）表示：如果当前地址大小属性是 16，则从 SI 寄存器读取偏移量；如果地址大小属性是 32，则从 ESI 寄存器读取。方括号也用于内存操作数，此时表示内存位置的内容是相对于段的偏移量。例如，[SRC] 表示源操作数的内容是相对于段的偏移量。A B; 表示将 B 的值赋给 A。符号 =、、 及 是用于比较两个值的关系型操作符，分别表示等于、不等于、大于或等于、小于或等于。如果 A 的值等于 B，则 A B 这样的关系型表达式为 TRUE；否则为 FALSE。表达式 "<< COUNT" 与 ">> COUNT" 表示根据计数操作数指出的位数，分别左移或右移目标操作数。

以下标识符会在算法说明中用到：

OperandSize 与 AddressSize - OperandSize 标识符代表指令的操作数大小属性：16 或 32 位。AddressSize 标识符代表地址大小属性：16 或 32 位。例如，以下伪代码表示操作数大小属性取决于所用 CMPS 指令的形式。

IF instruction CMPSW THEN OperandSize 16; ELSE IF instruction CMPSD THEN OperandSize 32; FI; FI; 如需有关如何确定这些属性的一般指导信息，请参阅“IA-32 英特尔(R) 体系结构软件开发人员手册”第 1 卷第 3 章中的“操作数大小与地址大小属性”。

StackAddrSize - 代表与指令关联的堆栈地址大小属性，其值为 16 或 32 位（请参阅“IA-32 英特尔(R) 体系结构软件开发人员手册”第 1 卷第 6 章中的“堆栈的地址大小属性”）。

SRC - 代表源操作数。

DEST - 代表目标操作数。

以下函数会在算法说明中用到：

ZeroExtend(value) - 返回根据指令的操作数大小属性进行零扩展的值。例如，如果操作数大小属性是 32，对字节值 -10 进行零扩展时，会将字节从 F6H 转换成双字值 000000F6H。如果传递给 ZeroExtend 函数的值的大小与操作数大小属性相同，则 ZeroExtend 返回未加修改的值。

SignExtend(value) - 返回根据指令的操作数大小属性进行符号扩展的值。例如，如果操作数大小属性是 32，对包含值 -10 的字节进行符号扩展时，会将字节从 F6H 转换成双字值 FFFFFFF6H。如果传递给 SignExtend 函数的值的大小与操作数大小属性相同，则 SignExtend 返回未加修改的值。

SaturateSignedWordToSignedByte - 将有符号 16 位值转换成有符号 8 位值。如果有符号 16 位值小于 -128，则使用饱和值 -128 (80H) 表示；如果大于 127，则使用饱和值 127 (7FH) 表示。

SaturateSignedDwordToSignedWord - 将有符号 32 位值转换成有符号 16 位值。如果有符号 32 位值小于 -32768，则使用饱和值 -32768 (8000H) 表示；如果大于 32767，则使用饱和值 32767 (7FFFH) 表示。

SaturateSignedWordToUnsignedByte - 将有符号 16 位值转换成无符号 8 位值。如果有符号 16 位值小于 0，则使用饱和值零 (00H) 表示；如果大于 255，则使用饱和值 255 (FFH) 表示。

SaturateToSignedByte - 使用有符号 8 位值表示操作结果。如果结果小于 -128，则使用饱和值 -128 (80H) 表示；如果大于 127，则使用饱和值 127 (7FH) 表示。

SaturateToSignedWord - 使用有符号 16 位值表示操作结果。如果结果小于 -32768，则使用饱和值 -32768 (8000H) 表示；如果大于 32767，则使用饱和值 32767 (7FFFH) 表示。

SaturateToUnsignedByte - 使用有符号 8 位值表示操作结果。如果结果小于 0，则使用饱和值零 (00H) 表示；如果大于 255，则使用饱和值 255 (FFH) 表示。

SaturateToUnsignedWord - 使用有符号 16 位值表示操作结果。如果结果小于 0，则使用饱和值零 (00H) 表示；如果大于 65535，则使用饱和值 65535 (FFFFH) 表示。LowOrderWord(DEST * SRC) - 将两个字操作数相乘，双字结果中的最低有效字存储到目标操作数。HighOrderWord(DEST * SRC) - 将两个字操作数相乘，双字结果中的最高有效字存储到目标操作数。

Push(value) - 将值压入堆栈。压入的字节数由指令的操作数大小属性确定。如需有关压入操作的详细信息，请参阅 PUSH - 将字或双字压入堆栈中的“操作”部分。

Pop() 从栈顶删除值并返回它。语句 EAX Pop(); 将栈顶的 32 位值赋给 EAX。Pop 将返回字或双字，具体取决于操作数大小属性。如需有关弹出操作的详细信息，请参阅 POP - 将值弹出堆栈中的“操作”部分。

PopRegisterStack - 将 FPU ST(0) 寄存器标记为空，并使 FPU 寄存器堆栈指针 (TOP) 递增 1。

Switch-Tasks - 执行任务切换。

Bit(BitBase, BitOffset) - 返回位串（内存或寄存器中的位序列）中某个位的值。位在寄存器与内存字节中按照从低位到高位的顺序编号。如果基址操作数是寄存器，偏移量可以在 0..31 范围内。此偏移量代表指定的寄存器中的某个位。例如，函数 Bit[EAX, 21] 图示如下：

BIT[EAX,21] 的位偏移量

如果 BitBase 是内存地址，则 BitOffset 可以在 -2 GBits 到 2 GBits 范围内。确定地址的位编号为地址 (BitBase + (BitOffset DIV 8)) 上的字节内的 (Offset MOD 8)，其中 DIV 是向负无穷大取整的有符号除法，MOD 返回正数。

英特尔 C++ 编译器等价内部函数是一些特殊的 C++ 编码扩展，它允许使用 C 函数调用与 C 变量的语法替代硬件寄存器。通过使用这些内部函数，开发人员可以不再需要管理寄存器与编写汇编程序。而且，编译器优化了指令调度，从而使可执行文件运行速度更快。
