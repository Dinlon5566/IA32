说明

将目标操作数与源操作数相加，和存储到目标位置。目标操作数总是 FPU 寄存器，源操作数可以是寄存器或内存位置。内存中的源操作数可以是单精度实数、双精度实数、字整数或短整数格式。

无操作数版的指令将 ST(0) 寄存器的内容加到 ST(1) 寄存器上。单操作数版将内存位置的内容（实数或整数值）加到 ST(0) 寄存器的内容上。双操作数版将 ST(0) 寄存器的内容加到 ST(0) 寄存器的内容上，或者反过来。ST(0) 中的值可以通过编写以下代码翻倍：

FADD ST(0), ST(0);

存储结果之后，FADDP 指令执行额外的操作，即弹出 FPU 寄存器堆栈。为了弹出寄存器堆栈，处理器将 ST(0) 寄存器标记为空，并使堆栈指针 (TOP) 递增 1。（无操作数版的浮点加法指令总是弹出寄存器堆栈。在某些汇编器中，此指令的助记符是 FADD，而不是 FADDP）。

执行加法之前，FIADD 指令将整型源操作数转换成扩展型实数格式。

下文的表格显示将各类数值相加时得到的结果（假设未发生上溢或下溢）。

符号相反的操作数之和为 0 时，结果为 +0，但在向 - 取整模式中除外，此时结果为 -0。源操作数为整数 0 时，它被当作 +0。

两个操作数是同符号的无穷大时，结果为 ，符号不变。如果两个操作数为符号相反的无穷大，则生成操作无效异常。

影响的 FPU 标志

C1 - 如果发生堆栈下溢，则设置为 0。

如果结果取整则设置为 1；否则清除。

 

C0、C2、C3 - 未定义。

浮点异常

#IS - 发生堆栈下溢。

#IA - 操作数是 SNaN 值或不支持的格式。操作数是不同符号的无穷大。

#D - 源操作数是非规格化值。

#U - 结果对于目标格式而言太小。#O - 结果对于目标格式而言太大。

#P - 值无法按目标格式精确表示。

保护模式异常

#GP(0) - 如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。如果 DS、ES、FS 或 GS 寄存器包含空的段选择器。

#SS(0) - 如果内存操作数有效地址超出 SS 段限制。

#NM - 如果 CR0 中的 EM 或 TS 设置为 1。

#PF(错误代码) - 如果发生页错误。

#AC(0) - 如果在当前特权级别为 3 且启用对齐检查的情况下进行未对齐的内存引用。

实地址模式异常

#GP - 如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。

#SS - 如果内存操作数有效地址超出 SS 段限制。

#NM - 如果 CR0 中的 EM 或 TS 设置为 1。

虚 8086 模式异常

#GP(0) - 如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。

#SS(0) - 如果内存操作数有效地址超出 SS 段限制。

#NM - 如果 CR0 中的 EM 或 TS 设置为 1。

#PF(错误代码) - 如果发生页错误。

#AC(0) - 如果在启用对齐检查的情况下进行未对齐的内存引用。
