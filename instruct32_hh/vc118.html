说明

FST 指令将 ST(0) 寄存器中的值复制到目标操作数，目标操作数可以是内存位置或 FPU 寄存器堆栈中的另一个寄存器。将值存储到内存时，值会转换成单精度或双精度实数格式。

FSTP 指令先执行同 FST 指令相同的操作，然后弹出寄存器堆栈。为了弹出寄存器堆栈，处理器将 ST(0) 寄存器标记为空，并使堆栈指针 (TOP) 递增 1。FSTP 指令还可以按扩展的实数格式在内存中存储值。

如果目标操作数是内存位置，则操作数指定要存储的目标值第一个字节的地址。如果目标操作数是寄存器，则操作数按相对于栈顶的方式指定寄存器堆栈中的一个寄存器。

如果目标大小是单精度或双精度实数，则要存储的值的有效位按目标的宽度取整（根据 FPU 控制字的 RC 字段指定的取整模式），指数会转换成目标格式的宽度与偏差。如果存储的值对于目标格式而言太大，则生成数值上溢异常 (#O)，并且如果未屏蔽该异常，则不会将任何值存储到目标操作数。如果要存储的值是非规格化值，则不生成非规格化异常 (#D)。此情况只当作数值下溢异常 (#U) 情况发出信号。

如果要存储的值是 +0、+ 或 NaN，则按适合目标格式的要求，截断有效位与指数的最低有效位。此操作可以保留值的 0、 或 NaN 标识。

如果目标操作数是非空的寄存器，则不生成操作无效异常。

影响的 FPU 标志

C1 - 如果发生堆栈下溢，则设置为 0。如果生成浮点不精确异常 (#P)，则表示取整方向：

0 非向上取整；1 向上取整。

C0、C2、C3 - 未定义。

浮点异常

#IS - 发生堆栈下溢。

#IA - 源操作数是 SNaN 值或不支持的格式。

#U - 结果对于目标格式而言太小。

#O - 结果对于目标格式而言太大。

#P - 值无法按目标格式精确表示。

保护模式异常

#GP(0) - 如果目标位于不可写的段。如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。如果 DS、ES、FS、或 GS 寄存器用于访问内存，并且它包含空的段选择器。

#SS(0) - 如果内存操作数有效地址超出 SS 段限制。

#NM - 如果 CR0 中的 EM 或 TS 设置为 1。

#PF(错误代码) - 如果发生页错误。

#AC(0) - 如果启用对齐检查并在当前特权级别为 3 时进行未对齐的内存引用。

实地址模式异常

#GP - 如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。

#SS - 如果内存操作数有效地址超出 SS 段限制。

#NM - 如果 CR0 中的 EM 或 TS 设置为 1。

虚 8086 模式异常

#GP(0) - 如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。

#SS(0) - 如果内存操作数有效地址超出 SS 段限制。

#NM - 如果 CR0 中的 EM 或 TS 设置为 1。

#PF(错误代码) - 如果发生页错误。

#AC(0) - 如果在启用对齐检查的情况下进行未对齐的内存引用。
