说明

检查 EFLAGS 寄存器中一个或多个状态标志（CF、OF、PF、SF 及 ZF）的状态，如果标志处于指定的状态（条件），则跳转到目标操作数指定的目标指令。条件代码 (cc) 与每条指令关联，用于指明测试条件。如果条件不满足，则不执行跳转操作，而是继续执行 Jcc 指令后面的指令。

目标指令使用相对偏移量（相对于 EIP 寄存器中指令指针当前值的有符号偏移量）指定。在汇编代码中，相对偏移量（rel8、rel16 或 rel32）通常指定为标签，但是在机器代码级别，它的编码形式是有符号的 8 位或 32 位立即数，并且会加到指令指针上。对于 -128 到 +127 的偏移量，指令编码的效率最高。如果操作数大小的属性为 16，EIP 寄存器的上面两个字节清 0，这样就产生了 16 位的最大指令指针。

每个 Jcc 助记符的条件在上文表格的“说明”列中给出。术语“小于”与“大于”用于有符号整数的比较；术语“高于”与“低于”用于无符号整数。

由于状态标志的特定状态有时存在两种含义，因此给某些操作码定义了两个助记符。例如，JA（高于时跳转）指令与 JNBE（低于或等于时跳转）指令是操作码 77H 的替代助记符。

Jcc 指令不支持远跳转（跳转到其它代码段）。条件跳转的目标在另一个段时，请使用 Jcc 指令的相反测试条件，然后使用无条件远跳转（JMP 指令）指令访问该段。例如，下面的条件远跳转是非法的：

JZ FARLABEL;

要实现此远跳转，请使用以下两条指令：

JNZ BEYOND;
JMP FARLABEL;
BEYOND:

JECXZ 与 JCXZ 指令不同于其它 Jcc 指令，它们不检查状态标志。相反，它们分别检查 ECX 与 CX 寄存器的内容是否为 0。CX 或 ECX 寄存器根据地址大小属性选择。在以条件循环指令（如 LOOPNE）终止的条件循环中，在开头处使用这些指令非常有价值。它们可以避免在 ECX 或 CX 寄存器等于 0 时进入循环，否则将导致循环分别执行 232 或 64K 次，而不是零次。

所有的条件跳转，不论跳转地址在哪里或能否缓存，都会转换成获取一个或两个缓存线的代码。

影响的标志

无。

保护模式异常

#GP(0) - 如果跳转目标的偏移量超出 CS 段的限制。

实地址模式异常

#GP - 如果跳转的目标偏移量超出 CS 段的限制，或是出现在 0 到 FFFFH 的有效地址空间之外。如果使用 32 位地址大小覆盖前缀，则可能会发生这种情况。

虚 8086 模式异常

#GP(0) - 如果跳转目标的偏移量超出 CS 段的限制，或是出现在 0 到 FFFFH 的有效地址空间之外。如果使用 32 位地址大小覆盖前缀，则可能会发生这种情况。
