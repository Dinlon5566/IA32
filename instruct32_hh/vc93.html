说明

将目标操作数除以源操作数，结果存储到目标位置。目标操作数（被除数）总是 FPU 寄存器，源操作数（除数）可以是寄存器或内存位置。内存中的源操作数可以是单精度实数、双精度实数、字整数或短整数格式。

无操作数版的指令将 ST(1) 寄存器的内容除以 ST(0) 寄存器的内容。单操作数版将 ST(0) 寄存器的内容除以内存位置的内容（实数或整数值）。双操作数版将 ST(0) 寄存器的内容除以 ST(i) 寄存器的内容，或者相反。

存储结果之后，FDIVP 指令执行额外的操作，即弹出 FPU 寄存器堆栈。为了弹出寄存器堆栈，处理器将 ST(0) 寄存器标记为空，并使堆栈指针 (TOP) 递增 1。无操作数版的浮点除法指令总是弹出寄存器堆栈。在某些汇编器中，此指令的助记符是 FDIV 而不是 FDIVP。

执行除法之前，FIDIV 指令将整型源操作数转换为成扩展型实数格式。源操作数为整数 0 时，它被当作 +0。

如果生成无掩码除零异常 (#Z)，则不存储任何结果；如果屏蔽该异常，则将带适当符号的 存储到目标操作数。

影响的 FPU 标志

C1 - 如果发生堆栈下溢，则设置为 0。如果生成不精确的结果异常 (#P)，则表示取整方向：0 非向上取整；1 向上取整。

C0、C2、C3 - 未定义。

浮点异常

#IS - 发生堆栈下溢。

#IA - 操作数是 SNaN 值或不支持的格式。

±∞ / ±∞; ±0 / ±0

#D - 源操作数是非规格化值。

#Z - DEST / ±0，其中 DEST 不等于 ±0。

#U - 结果对于目标格式而言太小。

#O - 结果对于目标格式而言太大。

#P - 值无法按目标格式精确表示。

保护模式异常

#GP(0) - 如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。如果 DS、ES、FS 或 GS 寄存器包含空的段选择器。

#SS(0) - 如果内存操作数有效地址超出 SS 段限制。

#NM - 如果 CR0 中的 EM 或 TS 设置为 1。

#PF(错误代码) - 如果发生页错误。

#AC(0) - 如果在当前特权级别为 3 且启用对齐检查的情况下进行未对齐的内存引用。

实地址模式异常

#GP - 如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。

#SS - 如果内存操作数有效地址超出 SS 段限制。

#NM - 如果 CR0 中的 EM 或 TS 设置为 1。
