说明

将数据从源操作数（第二个操作数）复制到目标操作数（第一个操作数）指定的 I/O 端口。源操作数为内存位置，其地址从 DS:EDI 或 DS:DI 寄存器中读取（具体取决于指令的地址大小属性，分别为 32 或 16）。（DS 段可以使用跨段前缀覆盖）。目标操作数是从 DX 寄存器中读取的 I/O 端口地址（从 0 到 65,535）。对于 8 位 I/O 端口，访问的 I/O 端口的大小（即源操作数与目标操作数的大小）由操作码确定；对于 16 或 32 位 I/O 端口，则由指令的操作数大小属性确定。

在汇编代码级别，此指令允许采用两种形式：“显式操作数”形式与“无操作数”形式。显式操作数形式（使用 OUTS 助记符指定）允许显式指定源操作数与目标操作数。这里，源操作数应该是表示 I/O 端口大小与源地址的符号，目标操作数必须是 DX。提供此显式操作数形式是为了便于记录；不过请注意，此形式提供的记录可能会令人误解。也就是说，源操作数符号必须指定操作数（字节、字或双字）的正确类型（大小），但不必指定正确的位置。位置总是由 DS:(E)SI 寄存器指定，在执行 OUTS 指令之前，必须先正确加载这些寄存器。

无操作数形式提供“简短形式”的字节、字及双字版 OUTS 指令。这里，同样假设 DS:(E)SI 是源操作数，假设 DX 是目标操作数。I/O 端口的大小通过选择助记符指定：OUTSB（字节）、OUTSW（字）或 OUTSD（双字）。

将字节、字或双字从内存位置传输到 I/O 端口之后，根据 EFLAGS 寄存器中的 DF 标志的设置，(E)SI 寄存器会自动递增或递减。（如果 DF 标志为 0，则 (E)SI 寄存器递增；如果 DF 标志为 1，则 (E)SI 寄存器递减）。对于字节操作，(E)SI 寄存器递增或递减 1；对于字操作，递增或递减 2；对于双字操作，递增或递减 4。

通过 REP 前缀，OUTS、OUTSB、OUTSW 及 OUTSD 指令可用于整块输入 ECX 字节、字或双字。如需有关 REP 前缀的说明，请参阅 REP/REPE/REPZ/REPNE/REPNZ - 重复字符串操作前缀。

此指令只能用于访问处理器 I/O 地址空间中的 I/O 端口。如需有关访问 I/O 地址空间中的 I/O 端口的详细信息，请参阅“IA-32 英特尔(R) 体系结构软件开发人员手册”第 1 卷第 9 章“输入/输出”。

英特尔(R) 体系结构兼容性

执行 OUTS、OUTSB、OUTSW 或 OUTSD 指令之后，奔腾® 处理器会在开始执行下一条指令之前，确保 EWBE# 引脚已通过采样激活。（请注意，如果 EWBE# 处于非活动状态，可以预取此指令，但是要到通过采用激活 EWBE# 引脚之后，才可以执行它）。只有奔腾处理器系列有 EWBE# 引脚；其它 IA-32 处理器都没有。对于奔腾 4 与 P6 系列处理器，在执行 OUTS、OUTSB、OUTSW 或 OUTSD 指令时，要到事务的数据阶段完成之后，处理器才会执行下一条指令。

操作

IF ((PE 1) AND ((CPL > IOPL) OR (VM 1)))
THEN (* Protected mode with CPL > IOPL or virtual-8086 mode *)
IF (Any I/O Permission Bit for I/O port being accessed 1)
THEN (* I/O operation is not allowed *)
#GP(0);
ELSE ( * I/O operation is allowed *)
DEST SRC; (* Writes to I/O port *)
FI;
ELSE (Real Mode or Protected Mode with CPL IOPL *)
DEST SRC; (* Writes to I/O port *)
FI;
IF (byte transfer)
THEN IF DF 0
THEN (E)SI (E)SI + 1;
ELSE (E)SI (E)SI - 1;
FI;
ELSE IF (word transfer)
THEN IF DF 0
THEN (E)SI (E)SI + 2;
ELSE (E)SI (E)SI - 2;
FI;
ELSE (* doubleword transfer *)
THEN IF DF 0
THEN (E)SI (E)SI + 4;
ELSE (E)SI (E)SI - 4;
FI;
FI;
FI;

影响的标志

无。

保护模式异常

#GP(0) - 如果 CPL 大于（特权较低）I/O 特权级别 (IOPL)，并且访问的 I/O 端口的 TSS 中相应的任何 I/O 权限位是 1。如果内存操作数的有效地址超出 CS、DS、ES、FS 或 GS 段的限制。如果段寄存器包含空的段选择器。

#PF(错误代码) - 如果发生页错误。

#AC(0) - 如果启用对齐检查并在当前特权级别为 3 时进行未对齐的内存引用。

实地址模式异常

#GP - 如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。

#SS - 如果内存操作数有效地址超出 SS 段限制。

虚 8086 模式异常

#GP(0) - 如果访问的 I/O 端口的 TSS 中的任何 I/O 权限位是 1。

#PF(错误代码) - 如果发生页错误。

#AC(0) - 如果在启用对齐检查的情况下进行未对齐的内存引用。
