UNPCKLPS - 扩展低位压缩单精度浮点值

操作码

指令

说明

0F 14 /r

UNPCKLPS xmm1, xmm2/m128

交叉组合 xmm1 与 xmm2/mem 低位四字中的单精度浮点值，然后将结果放入 xmm1。

将源操作数（第二个操作数）与目标操作数（第一个操作数）中的低位单精度浮点值展开并交叉组合。请参阅“图 3-21”。源操作数可以是 XMM 寄存器或 128 位内存位置；目标操作数是 XMM 寄存器。

图 3-21. UNPCKLPS 交叉组合操作

从内存操作数进行扩展时，具体实现的版本可能只获取适当的 64 位；不过，此时仍执行 16 字节边界对齐检查及正常的段检查。

SIMD 浮点异常

无。

保护模式异常

#GP(0) - CS、DS、ES、FS 或 GS 段中的内存操作数有效地址非法。如果内存操作数未对齐 16 字节边界，不论是哪一段。

#SS(0) - SS 段中的地址非法。

#PF(错误代码) - 页错误。

#NM - 如果 CR0 中的 TS 设置为 1。

#XM - 如果发生无掩码 SIMD 浮点异常，且 CR4 中的 OSXMMEXCPT 是 1。

#UD - 如果发生无掩码 SIMD 浮点异常，且 CR4 中的 OSXMMEXCPT 是 0。如果 CR0 中的 EM 设置为 1。如果 CR4 中的 OSFXSR 是 0。如果 CPUID 功能标志 SSE2 是 0。

实地址模式异常

#GP(0) - 如果内存操作数未对齐 16 字节边界，不论是哪一段。

中断 13 - 如果操作数的任何部分出现在从 0 到 0FFFFH 的有效地址空间之外。

#NM - 如果 CR0 中的 TS 设置为 1。

#XM - 如果发生无掩码 SIMD 浮点异常，且 CR4 中的 OSXMMEXCPT 是 1。

#UD - 如果发生无掩码 SIMD 浮点异常，且 CR4 中的 OSXMMEXCPT 是 0。如果 CR0 中的 EM 设置为 1。如果 CR4 中的 OSFXSR 是 0。如果 CPUID 功能标志 SSE2 是 0。
