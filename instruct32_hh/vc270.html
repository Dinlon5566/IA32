说明

按第二个操作数（计数操作数）指定的位数将第一个操作数（目标操作数）中的位移位（循环移位），结果存储到目标操作数。目标操作数可以是寄存器或内存位置；计数操作数是一个无符号整数，它可以是立即数或 CL 寄存器中的值。处理器通过对计数操作数中除 5 个最低有效位之外的其它所有位执行掩码处理，将计数限制为 0 到 31。

循环左移 (ROL) 与带进位循环左移 (RCL) 指令将除最高有效位之外的其它所有位移到更高有效位的位置，最高有效位则循环移到最低有效位的位置（请参阅“IA-32 英特尔(R) 体系结构软件开发人员手册”第 1 卷的“图 6-10”）。循环右移 (ROR) 与带进位循环右移 (RCL) 指令将除最低有效位之外的其它所有位移到更低有效位的位置，最低有效位则循环移到最高有效位的位置（请参阅“IA-32 英特尔(R) 体系结构软件开发人员手册”第 1 卷的“图 6-10”）。

RCL 与 RCR 指令在循环移位中包括 CF 标志。RCL 指令将 CF 标志移到最低有效位，并将最高有效位移到 CF 标志（请参阅“IA-32 英特尔(R)3q21`1 体系结构软件开发人员手册”第 1 卷的“图 6-10”）。RCR 指令将 CF 标志移到最高有效位，并将最低有效位移到 CF 标志（请参阅“IA-32 英特尔(R)3q21`1 体系结构软件开发人员手册”第 1 卷的“图 6-10”）。对于 ROL 与 ROR 指令，CF 标志的原始值不是结果的一部分，但 CF 标志会接收从一端移动到另一端的位的副本。

OF 标志仅在 1 位循环移位中有定义；在其它所有情况中它都未定义（零位循环移位除外，它不执行任何操作，即不影响任何标志）。对于循环左移，OF 标志设置为 CF 位（循环移位之后）与结果的最高有效位之间的“异或”。对于循环右移，OF 标志设置为结果的两个最高有效位之间的“异或”。

英特尔(R) 体系结构兼容性

8086 不对循环移位计数执行掩码处理。不过，其它所有的“英特尔(R) 体系结构”处理器（从 Intel(R) 286 处理器起）都会通过掩码处理，将循环移位计数限制成 5 位，这样最大计数就是 31。在所有的操作模式（包括虚 8086 模式）中，都会执行这样的掩码处理，以缩短指令的最大执行时间。

操作

(* RCL and RCR instructions *)
SIZE OperandSize
CASE (determine count) OF
SIZE 8: tempCOUNT (COUNT AND 1FH) MOD 9;
SIZE 16: tempCOUNT (COUNT AND 1FH) MOD 17;
SIZE 32: tempCOUNT COUNT AND 1FH;
ESAC;
(* RCL instruction operation *)
WHILE (tempCOUNT 0)
DO
tempCF MSB(DEST);
DEST (DEST * 2) + CF;
CF tempCF;
tempCOUNT tempCOUNT - 1;
OD;
ELIHW;
IF COUNT 1
THEN OF MSB(DEST) XOR CF;
ELSE OF is undefined;
FI;
(* RCR instruction operation *)
IF COUNT 1
THEN OF MSB(DEST) XOR CF;
ELSE OF is undefined;
FI;
WHILE (tempCOUNT 0)
DO
tempCF LSB(SRC);
DEST (DEST / 2) + (CF * 2SIZE);
CF tempCF;
tempCOUNT tempCOUNT - 1;
OD;
(* ROL and ROR instructions *)
SIZE OperandSize
CASE (determine count) OF
SIZE 8: tempCOUNT COUNT MOD 8;
SIZE 16: tempCOUNT COUNT MOD 16;
SIZE 32: tempCOUNT COUNT MOD 32;
ESAC;
(* ROL instruction operation *)
WHILE (tempCOUNT 0)
DO
tempCF MSB(DEST);
DEST (DEST * 2) + tempCF;
tempCOUNT tempCOUNT - 1;
OD;
ELIHW;
CF LSB(DEST);
IF COUNT 1
THEN OF MSB(DEST) XOR CF;
ELSE OF is undefined;
FI;
(* ROR instruction operation *)
WHILE (tempCOUNT 0)
DO
tempCF LSB(SRC);
DEST (DEST / 2) + (tempCF * 2SIZE);
tempCOUNT tempCOUNT - 1;
OD;
ELIHW;
CF MSB(DEST);
IF COUNT 1
THEN OF MSB(DEST) XOR MSB - 1(DEST);
ELSE OF is undefined;
FI;

影响的标志

CF 标志包含移到其中的位的值。OF 标志仅受一位循环移位的影响（请参阅上面的“说明”）；它在多位循环移位中未定义。SF、ZF、AF 及 PF 标志不受影响。

保护模式异常

#GP(0) - 如果源操作数位于不可写的段。如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。如果 DS、ES、FS 或 GS 寄存器包含空的段选择器。

#SS(0) - 如果内存操作数有效地址超出 SS 段限制。

#PF(错误代码) - 如果发生页错误。

#AC(0) - 如果启用对齐检查并在当前特权级别为 3 时进行未对齐的内存引用。

实地址模式异常

#GP - 如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。

#SS - 如果内存操作数有效地址超出 SS 段限制。

虚 8086 模式异常

#GP(0) - 如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。

#SS(0) - 如果内存操作数有效地址超出 SS 段限制。

#PF(错误代码) - 如果发生页错误。

#AC(0) - 如果在启用对齐检查的情况下进行未对齐的内存引用。
