MOVSHDUP - 移动压缩单精度浮点值高位并复制

 

指令

说明

 

MOVSHDUP xmm1, xmm2/m128

将 XMM2/Mem 中代表压缩 SP 数据元素的 128 位移到 XMM1 寄存器，并复制高位部分。

说明

线性地址对应于引用的内存数据的最低有效字节。指出内存地址时，将加载内存位置 m128 处 16 个字节的数据，并复制位置 1 与 3 处的单精度元素。使用寄存器-寄存器形式的此操作时，将执行相同的操作，但数据来自 128 位源寄存器。

操作

 if (source == m128) {

// load instruction

xmm1[31-0] = m128[63-32];

xmm1[63-32] = m128[63-32];

xmm1[95-64] = m128[127-96];

xmm1[127-96] = m128[127-96];

}

else {

// move instruction

xmm1[31-0] = xmm2[63-32];

xmm1[63-32] = xmm2[63-32];
xmm1[95-64] = xmm2[127-96];

xmm1[127-96] = xmm2[127-96];

}

异常

如果未对齐 16 字节边界，则产生一般保护性异常，无论段如何。

数值异常

无

保护模式异常

#GP(0) - CS、DS、ES、FS 或 GS 段中的内存操作数有效地址非法。

#SS(0) - SS 段中的地址非法。

#PF(错误代码) - 页错误。

#NM - 如果 CR0 中的 TS 设置为 1。

#UD - 如果 CR0.EM = 1。如果 CR4.OSFXSR(bit 9) = 0。如果 CPUID.PNI(ECX bit 0) = 0。MOVSHDUP：移动压缩单精度浮点值高位并复制（续）
