说明

从第二个操作数（源操作数）将远指针（段选择器与偏移量）加载到段寄存器与第一个操作数（目标操作数）。源操作数指定内存中的 48 位或 32 位指针，具体取决于操作数大小属性的当前设置（分别为 32 位或 16 位）。指令操作码与目标操作数指定段寄存器/通用寄存器对。源操作数中的 16 位段选择器加载到操作码指定的段寄存器（DS、SS、ES、FS 或 GS）。32 位或 16 位偏移量加载到目标操作数指定的寄存器。

如果在保护模式中执行其中一条指令，则将源操作数中段选择器指向的段描述符中的其它信息加载到所选段寄存器的隐藏部分。

还是在保护模式中，空选择器（值是 0000 到 0003）可以加载到 DS、ES、FS 或 GS 寄存器，而不会导致保护异常。（随后引用段寄存器中加载空选择器的段时，将导致一般保护性异常 (#GP)，并且不发生该段的内存引用）。

操作

IF ProtectedMode
THEN IF SS is loaded
THEN IF SegementSelector null
THEN #GP(0);
FI;
ELSE IF Segment selector index is not within descriptor table limits
OR Segment selector RPL CPL
OR Access rights indicate nonwritable data segment
OR DPL CPL
THEN #GP(selector);
FI;
ELSE IF Segment marked not present
THEN #SS(selector);
FI;
SS SegmentSelector(SRC);
SS SegmentDescriptor([SRC]);
ELSE IF DS, ES, FS, or GS is loaded with non-null segment selector
THEN IF Segment selector index is not within descriptor table limits
OR Access rights indicate segment neither data nor readable code segment
OR (Segment is data or nonconforming-code segment
AND both RPL and CPL > DPL)
THEN #GP(selector);
FI;
ELSE IF Segment marked not present
THEN #NP(selector);
FI;
SegmentRegister SegmentSelector(SRC) AND RPL;
SegmentRegister SegmentDescriptor([SRC]);
ELSE IF DS, ES, FS, or GS is loaded with a null selector:
SegmentRegister NullSelector;
SegmentRegister(DescriptorValidBit) 0; (*hidden flag; not accessible by software*)
FI;
FI;
IF (Real-Address or Virtual-8086 Mode)
THEN
SegmentRegister SegmentSelector(SRC);
FI;
DEST Offset(SRC);

影响的标志

无。

保护模式异常

#UD - 如果源操作数不是内存位置。

#GP(0) - 如果空选择器加载到 SS 寄存器。如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。如果 DS、ES、FS、或 GS 寄存器用于访问内存，并且它包含空的段选择器。

#GP(选择器) - 如果给 SS 寄存器加载数据，并且以下任何一个条件为真：段选择器索引超出描述符表格限制；段选择器 RPL 不等于 CPL；所选的段不是可写的数据段；或者是 DPL 不等于 CPL。如果将非空的段选择器加载到 DS、ES、FS 或 GS 寄存器，并且以下任何一个条件为真：段选择器索引超出描述符表格限制；所选的段既不是数据段，也不是可读的代码段；或所选的段是数据段或非相容代码段，但 RPL 与 CPL 都大于 CPL。

#SS(0) - 如果内存操作数有效地址超出 SS 段限制。

#SS(选择器) - 如果给 SS 寄存器加载数据，并且段标记成不存在。

#NP(选择器) - 如果将非空的段选择器加载到 DS、ES、FS 或 GS 寄存器，并且段标记成不存在。

#PF(错误代码) - 如果发生页错误。

#AC(0) - 如果启用对齐检查并在当前特权级别为 3 时进行未对齐的内存引用。

实地址模式异常

#GP - 如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。

#SS - 如果内存操作数有效地址超出 SS 段限制。

#UD - 如果源操作数不是内存位置。

虚 8086 模式异常

#UD - 如果源操作数不是内存位置。

#GP(0) - 如果内存操作数有效地址超出 CS、DS、ES、FS 或 GS 段限制。

#SS(0) - 如果内存操作数有效地址超出 SS 段限制。

#PF(错误代码) - 如果发生页错误。

#AC(0) - 如果在启用对齐检查的情况下进行未对齐的内存引用。
