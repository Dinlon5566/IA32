MOVUPS - 移动未对齐的压缩单精度浮点值

操作码

指令

说明

0F,10 /r

MOVUPS xmm1, xmm2/m128

将压缩单精度浮点值从 xmm2/m128 移到 xmm1。

0F,11 /r

MOVUPS xmm2/m128, xmm1

将压缩单精度浮点值从 xmm1 移到 xmm2/m128。

说明

从源操作数（第二个操作数）将包含四个压缩单精度浮点值的双四字移到目标操作数（第一个操作数）。此指令可用于将 128 位内存位置的内容加载到 XMM 寄存器、将 XMM 寄存器的内容存储到 128 位内存位置，或是在两个 XMM 寄存器之间移动数据。源操作数或目标操作数是内存操作数时，操作数可以不对齐 16 字节边界，并且不会导致生成一般保护性异常 (#GP)。

对于已知对齐 16 字节边界的内存位置，要在其中移入/移出压缩单精度浮点值，请使用 MOVAPS 指令。

在 16 位寻址模式中执行时，不允许 128 位数据访问的线性地址与 16 位段尾重叠，此时的行为定义成保留行为。在这种情形下，特定的处理器版本可能会生成 #GP 异常，也可能不会，并且，跨越段尾的地址可能会通过舍位变成段首地址，也可能不会。

操作

DEST SRC;
* #GP if SRC or DEST unaligned memory operand *;

SIMD 浮点异常

无。

保护模式异常

#GP(0) - CS、DS、ES、FS 或 GS 段中的内存操作数有效地址非法。

#SS(0) - SS 段中的地址非法。

#PF(错误代码) - 页错误。

#NM - 如果 CR0 中的 TS 设置为 1。

#XM - 如果发生无掩码 SIMD 浮点异常，且 CR4 中的 OSXMMEXCPT 是 1。

#UD - 如果发生无掩码 SIMD 浮点异常，且 CR4 中的 OSXMMEXCPT 是 0。如果 CR0 中的 EM 设置为 1。如果 CR4 中的 OSFXSR 是 0。如果 CPUID 功能标志 SSE 是 0。

实地址模式异常

中断 13 - 如果操作数的任何部分出现在从 0 到 0FFFFH 的有效地址空间之外。

#NM - 如果 CR0 中的 TS 设置为 1。

#XM - 如果发生无掩码 SIMD 浮点异常，且 CR4 中的 OSXMMEXCPT 是 1。

#UD - 如果发生无掩码 SIMD 浮点异常，且 CR4 中的 OSXMMEXCPT 是 0。如果 CR0 中的 EM 设置为 1。如果 CR4 中的 OSFXSR 是 0。如果 CPUID 功能标志 SSE 是 0。

虚 8086 模式异常

与“实地址模式”中的异常相同。
