自我修改代码导致的管道清除

管道由于自我修改代码问题而清除，对性能造成了负面影响。
对于采用“超线程技术”的系统或多处理器系统，估计的影响代表总计处理器时间影响（系统上所有逻辑/物理处理器之和），而不是“实际执行”时间影响。因此，在采用“超线程技术”的系统或多处理器系统上，很可能会看到细节的时间影响大于对负载实际执行时间的影响。请注意，在单处理器系统上，处理器时间与实际执行时间相同。

 
 
计数器相关性：

此细节与以下性能计数器函数相关：

SelfModifyingCodeClear 性能影响 = ((自我修改代码清除*1000)/时钟信号)*100
较低值：0.2
较高值：2

在“SelfModifyingCodeClear 性能影响”较高时，此细节具有实质性意义。

建议：
  • 避免自我修改代码管道清除
