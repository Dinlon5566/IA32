ESCR 限制导致的与线程无关 (AT-E)

此事件特定于逻辑处理器。不过，由于 ESCR 的限制，在同一次运行中无法同时在两个逻辑处理器上采样，所以此事件编程为与线程无关。测量的样本计数是物理处理器封装上发生的事件总数。与线程无关事件的事件中断在逻辑处理器 0 上设置。整个系统的样本计数就物理处理器封装而言是准确的。不过，进程、模块、函数或源代码的特定 IP 地址处的样本计数可能不准确。这是因为逻辑处理器 1 上特定 IP 地址处的样本计数将计在逻辑处理器 0 上，而逻辑处理器 0 非常可能在其它的 IP 地址上。逻辑处理器 0 上的 IP 地址记录为事件的位置。如果使用 AT-E 事件，查看系统级以下的数据时，请务必注意这一点。
