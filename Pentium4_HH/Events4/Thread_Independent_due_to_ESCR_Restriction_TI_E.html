ESCR 限制導致的與執行緒無關 (AT-E)

此事件特定於邏輯處理器。不過，由於 ESCR 的限制，在同一次執行中無法同時在兩個邏輯處理器上採樣，所以此事件程式設計為與執行緒無關。測量的樣本計數是物理處理器封裝上發生的事件總數。與執行緒無關事件的事件中斷在邏輯處理器 0 上設定。整個系統的樣本計數就物理處理器封裝而言是準確的。不過，程序、模組、函式或原始碼的特定 IP 地址處的樣本計數可能不準確。這是因為邏輯處理器 1 上特定 IP 地址處的樣本計數將計在邏輯處理器 0 上，而邏輯處理器 0 非常可能在其它的 IP 地址上。邏輯處理器 0 上的 IP 地址記錄爲事件的位置。如果使用 AT-E 事件，檢視系統級以下的數據時，請務必注意這一點。
