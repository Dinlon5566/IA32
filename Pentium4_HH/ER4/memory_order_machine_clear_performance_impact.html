此比率的值较高时，表示管道由于内存顺序问题而清除，并对性能造成了负面影响。这通常表示错误共享。两个线程访问相同缓存线中不同的或独立的数据时，将发生错误共享。以下情况会导致错误共享：

  • 情况 1：在相同的缓存线上同时存在一个线程的同步锁与另一个线程的线程专用变量。一个线程修改同步变量时，缓存线变成“污损”状态，并且必须写到每个逻辑或物理处理器。在采用“超线程技术”的系统上，这会导致缓存线收回，以保持缓存一致性。在多处理器系统上，这会导致内存顺序机器清除冲突。这两种系统都有与它们关联的显著性能惩罚。

  • 情况 2：两个没有同步的线程之间存在正确共享。这是程序错误，并且也会导致显著的性能惩罚。
