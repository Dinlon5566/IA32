此比率的值較高時，表示管道由於記憶體順序問題而清除，並對效能造成了負面影響。這通常表示錯誤共享。兩個執行緒訪問相同快取線中不同的或獨立的數據時，將發生錯誤共享。以下情況會導致錯誤共享：

  • 情況 1：在相同的快取線上同時存在一個執行緒的同步鎖與另一個執行緒的執行緒專用變數。一個執行緒修改同步變數時，快取線變成“污損”狀態，並且必須寫到每個邏輯或物理處理器。在採用“超執行緒技術”的系統上，這會導致快取線收回，以保持快取一致性。在多處理器系統上，這會導致記憶體順序機器清除衝突。這兩種系統都有與它們關聯的顯著效能懲罰。

  • 情況 2：兩個沒有同步的執行緒之間存在正確共享。這是程式錯誤，並且也會導致顯著的效能懲罰。
