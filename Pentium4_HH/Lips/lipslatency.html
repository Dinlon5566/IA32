延迟

Latency 表示可以将一条指令替换成总体延迟更短的其它指令序列。

建议

尝试将此指令替换成可以在更少的周期内执行相同操作的其它指令序列。

下表列出会导致 Latency 惩罚的指令，并提供有关如何尽量降低惩罚的建议。

指令

延迟说明

建议

inc 与 dec

inc 与 dec 指令只修改标志寄存器中的一部分位。这会导致对标志寄存器上所有先前写入的依赖性。因为这些指令用于更改加载（其它许多指令都依赖于此操作）的地址，在它们处于关键路径之上时，这尤其会成为问题。

将 inc 与 dec

指令替换成 add 或 sub 指令，因为 add 与 sub 会覆盖所有标志位。

 

shift

同以前各代处理器相比，shift 指令在奔腾 4 处理器上的延迟更长。固定与可变移位的延迟相同。

add 序列的延迟比三位及以下的左移延迟更短。

如果 shift 在关键路径上，请将它替换成 add（最多三条）序列。如果它的延迟并不关键，请仍使用 shift，原因在于它产生的微操作更少。

rotate

同以前各代处理器相比，rotate 指令在奔腾 4 处理器上的延迟更长。  

按立即数与寄存器的循环移位比普通移位指令的开销更大。循环移动 1 位与普通移位的延迟相同。

避免使用按寄存器或立即数循环移位指令。如果可能，请替换成循环移动 1 位的指令。

 

mul 与 imul

整数乘法运算 mul 与 imul 在浮点单元中执行，因此这些指令不应该同浮点指令并行执行。由于在浮点单元中执行，它们还会导致一些额外的延迟。如果上一个周期执行 fmul，则浮点乘法指令 (fmul) 将延迟一个周期。乘法器只能每隔一个周期接受一对新操作数。

将乘以小常数的整数乘法运算替换成两条或更多 add 与 lea 指令，在这些乘法运算是相关链的一部分时，尤其应该这样做。
