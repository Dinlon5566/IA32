浮点相关

FP_Dep 表示一种惩罚状态。

为其发出 FP_Dep 的指令依赖浮点管道中当前执行的指令的输出。因此它会一直暂停到浮点指令完成执行。

建议

大多数浮点指令在开始执行时，都要求其操作数在栈顶 (TOS)。不过，浮点指令在执行其延迟周期时，它的操作数不一定要在 TOS 中。

使用 FXCH 指令交换此指令与正在执行的浮点指令的操作数在堆栈中的位置。这样，此指令就可以立即开始执行。请注意，此指令交换目标寄存器与栈顶 (TOS) 寄存器的内容。如果未显式指定目标寄存器，则使用 st(1)。

FADD C（指令 2）有 3 个时钟周期的执行延迟。FADD D（指令 3）暂停，等待“指令 2”完成执行。同样，在前面的指令完成在 TOS 上的操作之前，其它指令都无法开始执行。
执行时间总计：10 个周期

通过重新组织代码，插入 FXCH 指令以交换 TOS 的内容，可以隐藏浮点指令的执行延迟。例如，跟在“指令 5”之后的 FXCH 指令将 B + C 放入 TOS。因此，即使在“指令 6”继续执行（在 ST(1) 上操作）期间，用于将 D 加到 B + C 上的“指令 3”仍可以立即执行。
执行时间总计：6 个周期
