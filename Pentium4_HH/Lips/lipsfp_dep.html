浮點相關

FP_Dep 表示一種懲罰狀態。

為其發出 FP_Dep 的指令依賴浮點管道中目前執行的指令的輸出。因此它會一直暫停到浮點指令完成執行。

建議

大多數浮點指令在開始執行時，都要求其運算元在棧頂 (TOS)。不過，浮點指令在執行其延遲週期時，它的運算元不一定要在 TOS 中。

使用 FXCH 指令交換此指令與正在執行的浮點指令的運算元在堆疊中的位置。這樣，此指令就可以立即開始執行。請注意，此指令交換目標暫存器與棧頂 (TOS) 暫存器的內容。如果未顯式指定目標暫存器，則使用 st(1)。

FADD C（指令 2）有 3 個時鐘週期的執行延遲。FADD D（指令 3）暫停，等待“指令 2”完成執行。同樣，在前面的指令完成在 TOS 上的操作之前，其它指令都無法開始執行。
執行時間總計：10 個週期

通過重新組織程式碼，插入 FXCH 指令以交換 TOS 的內容，可以隱藏浮點指令的執行延遲。例如，跟在“指令 5”之後的 FXCH 指令將 B + C 放入 TOS。因此，即使在“指令 6”繼續執行（在 ST(1) 上操作）期間，用於將 D 加到 B + C 上的“指令 3”仍可以立即執行。
執行時間總計：6 個週期
