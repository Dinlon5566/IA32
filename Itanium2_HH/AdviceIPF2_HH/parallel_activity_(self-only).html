並行活動（僅自身）

此統計數據代表程式碼區域與自身並行操作的處理器時間百分比。它的計算方式是：目的碼區域在每個時間格中的時鐘訊號樣本數之和，除以目標區域中可能發生的總計時鐘訊號樣本數。簡單地說，它是僅限所選程式碼區域的實際 並行工作除以可能的 並行工作。
下圖使用三個不同程式碼區域（藍色、綠色及紅色）的隨時間時鐘訊號樣本的簡單表示來闡述這個概念。水平軸代表時間，每個時間格的寬度代表時鐘訊號樣本之間的差異（採樣頻率值）。垂直軸代表系統上的邏輯處理器 (CPU)。請注意，這可以是每個處理器都採用“超執行緒技術”的四處理器系統或雙處理器系統。假設藍色程式碼區域是感興趣的區域（所選的程式碼區域），則可以檢查 t1 與 t7 這兩個不同的時間格，並確定自身並行活動。由於藍色程式碼區域是感興趣的區域，我們希望瞭解何時藍色區域的工作在多處理器上同時發生（也就是，在相同的時間格中，大於一個藍色區域）。在時間格 t1 中，藍色程式碼區域在 CPU 0 與 2 上有時鐘訊號樣本。使用 CPU 0 作為參考處理器來判斷並行化時，其它三個處理器在藍色區域有發生並行工作的“可能性”。因此，可能的 藍色區域並行工作是 3 個時鐘訊號樣本。對於 t1，因為其它 CPU 中只有一個 (CPU 2) 有藍色程式碼區域的時鐘訊號樣本，所以發生的實際 藍色區域工作是 1 個時鐘訊號樣本。這表示，對於時間格 t1，並行活動等於 1/3 或 33%。使用相同的演算法檢查時間格 t7。與 t1 一樣，一個 CPU 必須執行藍色區域程式碼，其餘的三個處理器可能會執行藍色區域程式碼。因此，可能的 藍色區域並行工作不會改變，仍是 3 個時鐘訊號樣本。在時間格 t7 中，有三個處理器在執行藍色區域程式碼，因此可以使用任何一個作為參考 CPU。因為其餘兩個 CPU 有藍色區域的時鐘訊號樣本，所以實際的 並行藍色區域工作是 2 個時鐘訊號樣本。因此，對於時間格 t7，並行活動等於 2/3 或 67%。請注意，自身並行活動計算僅包含所選程式碼區域與自身並行執行的時間，不包括其它程式碼區域的活動。使用圖中整個負載的計算是：
自身並行活動 = 4/18 = 22%

備註

  • 實際並行藍色區域工作 = 4 個時鐘訊號樣本
    對於每個時間格，我們將 1 個藍色區域用作參考，將其餘的藍色區域相加：t1 = 1、t3 = 1、t6 = 0、t7 = 2、t9 = 0、t12 = 0（總計 4 個）
  • 可能的藍色區域並行工作 = 6 * 3 = 18 個時鐘訊號樣本
    使用藍色程式碼區域的時間格數 = 6（t1、t3、t6、t7、t9 及 t12）
    可用於並行工作的處理器數 = 3。（系統上有 4 個處理器，其中一個必須執行藍色程式碼區域，其餘 3 個處理器可能會處理並行工作。）
  • 即使分析整個負載，也只有執行藍色程式碼區域的時間格才是大家感興趣的：t1、t3、t6、t7、t9、t12（總計 6 個）。


計數器相關性：

此細節依賴於以下效能計數器函式：

自身並行活動 = 100*(時鐘訊號樣本 - 活動時間格)/(活動時間格*(CPU 數-1))
較低值：0
較高值：0.1

在“自身並行活動”較高時，此細節具有實質性意義。
