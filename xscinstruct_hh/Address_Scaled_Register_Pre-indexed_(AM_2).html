調整的暫存器事先變址 (AM 2)

語法

[basereg, +/-indexreg, LSL #sh_imm]!
| [basereg, +/-indexreg, LSR #sh_imm]!
| [basereg, +/-indexreg, ASR #sh_imm]!
| [basereg, +/-indexreg, ROR #sh_imm]!
| [basereg, +/-indexreg, RRX]!

其中：

 basereg

指定包含基址的暫存器。

 indexreg

指定包含可加到 basereg 或從中減去的值的暫存器。

 LSL

指定邏輯左移。

 LSR

指定邏輯右移。

 ASR

指定算術右移。

 ROR

指定循環右移。

 RRX

指定擴充套件循環右移。

 !

設定 W 位，以更新基址暫存器

 
sh_imm 的值指定移位或循環移位寬度，它取決於指令：

指令

寬度

 LSL

0 到 31，在 sh_imm 欄位中編碼。

 LSR

1 到 32。移位量 32 使用 sh_imm==0 編碼表示。其它移位量直接編碼。

 ASR

1 到 32。移位量 32 使用 sh_imm==0 編碼表示。其它移位量直接編碼。

 ROR

1 到 31，在 sh_imm 欄位中編碼。sh_imm == 0 編碼指定 RRX 選項。

說明

定址模式“調整的暫存器事先變址”執行以下操作（偽程式碼說明）：

case shift of
0b00 /* LSL */
index = indexreg Logical_Shift_Left sh_imm
0b01 /* LSR */
if sh_imm == 0 then /* LSR #32 */
   index = 0
else
   index = indexreg Logical_Shift_Right sh_imm
0b10 /* ASR */
if sh_imm == 0 then /* ASR #32 */
   if indexreg[31] == 1 then
       index = 0xFFFFFFFF
   else
       index = 0
else
   index = indexreg Arithmetic_Shift_Right
sh_imm
0b11 /* ROR or RRX */
if sh_imm == 0 then /* RRX */
   index = (C Flag Logical_Shift_Left 31) OR
             indexreg Logical_Shift_Right 1)
else /* ROR */
   index = indexreg Rotate_Right sh_imm
end case
if U == 1 then
   addr = basereg + index
else /* U == 0 */
   addr = basereg - index
if ConditionPassed (condition) then
   basereg = addr

移位的或循環移位的變址暫存器 indexreg 的值加到基址暫存器 basereg 的值，或從中減去。如果指令中指定的條件與條件程式碼狀態匹配，則它寫回基址暫存器 basereg。

B 位表示無符號位元組 ( B == 1 ) 或字訪問 ( B == 0 )。

L 位表示“載入”( L == 1 ) 或“儲存”( L == 0 ) 指令。

 注意

如果 R15 指定為暫存器 basereg 或 indexreg，則結果將會無法預知。
為 basereg 與 indexreg 指定相同的暫存器時，結果將會無法預知。

示例

LDR    R8, [R6, R9, LSR #1]!
