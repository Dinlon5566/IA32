立即数 (AM 1)

语法

#imm

其中：

 imm

将指令中编码的常数操作数指定为 8 位立即数 ( imm_8 ) 与 4 位立即数 ( rt_imm )。因此，imm 与将 imm_8 循环右移 ( 2*rt_imm ) 位的结果相同。

说明

数据处理操作数“立即数”执行以下操作（伪代码说明）：

addr_mode = imm_8 Rotate_Right (rt_imm *2)
if rt_imm == 0 then
 shifter_carry_out = C flag
else /* rt_imm != 0 */
 shifter_carry_out = addr_mode[31]

此数据处理操作数是指令中定义的常数操作数。addr_mode 的值是由 8 位立即数循环右移产生的。8 位立即数循环移位到 32 位字中任何偶数位的位置。如果循环移位立即数是零，则移位器的进位是 C 标志的值，否则它设置为 addr_mode 值的位 [31]。

 备注

对于此格式而言，唯一有效的 32 位立即数是可以通过将 8 位立即数循环右移偶数位后得到的那些立即数。
对 imm 的有些值，可以有多种编码。如果可以使用多个编码，则汇编器按照以下方式选择正确的那个：
如果 imm 介于 0 与 0xFF 之间，则使用带 rt_imm == 0 的编码。汇编器必须选择该编码。选择另一种编码将影响有些指令设置的 C 标志。
建议选择 rt_imm 值最小的编码。此选择不影响指令功能。

更精确的编码控制是使用语法直接指定的指令字段提供的：

#imm_8, rt_amount

其中：

rt_amount = 2*rt_imm

示例

ADC   R0, R11, #1
