立即數事後變址 (AM 2)

語法

[basereg], #+/-value

其中：

 basereg

指定包含基址的暫存器。

 value

指定 12 位立即數偏移量，與 basereg 的值一起使用，以形成地址。

說明

定址模式“立即數事後變址”執行以下操作（偽程式碼說明）：

addr = basereg
if ConditionPassed (condition) then
if U == 1 then
   basereg = basereg + value
else/* U == 0 */
   basereg = basereg - value

此定址模式使用基址暫存器 basereg 的值作為記憶體訪問地址。

如果指令中指定的條件與條件程式碼狀態匹配，則立即數偏移量的值會加到基址暫存器 basereg 的值，或從中減去。然後它寫回 basereg。

對於帶自動更新指針值的陣列的指針訪問，此定址模式非常有用。

B 位表示無符號位元組 ( B==1 ) 或字訪問 ( B==0 )。

L 位表示“載入”( L==1 ) 或“儲存”( L==0 ) 指令。

備註

指令 LDRBT、LDRT、STRBT 及 STRT 僅支援事後變址定址模式。對於這些指令，上述位模式會作如下修改：
bit[21]（W 位）是 1，而不是上述的 0。

對於僅支援事後變址定址模式（ LDRBT、LDRT、STRBT 或 STRT）的指令，語法 [basereg] 視作 [basereg], #0 的縮寫。對於其它所有指令，語法 [basereg] 不得視作 [basereg], #0 的縮寫。

注意

如果 R15 指定為暫存器 basereg，則結果將會無法預知。

示例

LDR    R9, [R7], #5
