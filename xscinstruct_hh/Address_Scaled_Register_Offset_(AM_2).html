调整的寄存器偏移量 (AM 2)

语法

  [basereg, +/-indexreg, LSL #sh_imm]
| [basereg, +/-indexreg, LSR #sh_imm]
| [basereg, +/-indexreg, ASR #sh_imm]
| [basereg, +/-indexreg, ROR #sh_imm]
| [basereg, +/-indexreg, RRX]

其中：

 basereg

指定包含基址的寄存器。

 indexreg

指定包含可加到 basereg 或从中减去的值的寄存器。

 LSL

指定逻辑左移。

 LSR

指定逻辑右移。

 ASR

指定算术右移。

 ROR

指定循环右移。

 RRX

指定扩展循环右移。

 
sh_imm 的值指定移位或循环移位宽度，它取决于指令：

指令

宽度

 LSL

0 到 31，在 sh_imm 字段中编码。

 LSR

1 到 32。移位量 32 使用 sh_imm==0 编码表示。其它移位量直接编码。

 ASR

1 到 32。移位量 32 使用 sh_imm==0 编码表示。其它移位量直接编码。

 ROR

1 到 31，在 sh_imm 字段中编码。sh_imm == 0 编码指定 RRX 选项。

说明

寻址模式“调整的寄存器偏移量”执行以下操作（伪代码说明）：

case shift of
0b00 /* LSL */
 index = indexreg Logical_Shift_Left sh_imm
0b01 /* LSR */
 if sh_imm == 0 then /* LSR #32 */
     index = 0
 else
     index = indexreg Logical_Shift_Right sh_imm
0b10 /* ASR */
 if sh_imm == 0 then /* ASR #32 */
     if indexreg[31] == 1 then
         index = 0xFFFFFFFF
     else
         index = 0
 else
     index = indexreg Arithmetic_Shift_Right
 sh_imm
0b11 /* ROR or RRX */
 if sh_imm == 0 then /* RRX */
     index = (C Flag Logical_Shift_Left 31) OR
              indexreg Logical_Shift_Right 1)
 else /* ROR */
     index = indexreg Rotate_Right sh_imm
end case
if U == 1 then
  addr = basereg + index
else /* U == 0 */
  addr = basereg - index

移位的或循环移位的变址寄存器 indexreg 的值加到基址寄存器 basereg 的值，或从中减去。

这些寻址模式用于访问比字节大的数组中的单个元素。

B 位表示无符号字节 ( B==1 ) 或字访问 ( B==0 )。

L 位表示“加载”( L==1 ) 或“存储”( L==0 ) 指令。

 注意

如果 R15 指定为寄存器 basereg，则使用的值是当前指令的地址加 8。如果 R15 指定为寄存器 indexreg，则结果将会无法预知。

示例

LDR    R5, [R3, R6, LSL #3]
