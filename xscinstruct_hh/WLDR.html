WLDR - 加载 WMMX 寄存器

语法

WLDRlength[condition] destD, address_mode
WLDR destC, address_mode

其中：
length 所需的数据长度，可以由以下字符之一指定：
B - 加载 8 位字节数据
H - 加载 16 位半字数据
W - 加载 32 位字数据
D - 加载 64 位双字数据
condition 可选。16 个条件之一。请参阅条件执行
destD 目标，“英特尔(R) 无线 MMX™”技术数据寄存器
destC 目标，“英特尔(R) 无线 MMX™”技术控制寄存器
address_mode 地址模式 5 - 加载与存储协处理器

说明

执行从内存到数据寄存器 DestD 或控制寄存器 DestC 的加载。
数据寄存器中可以加载 8、16、32 及 64 位数据值。
控制寄存器中只能加载 32 位值。
可加载的内存地址由寻址模式参数确定。
所有加载的数据均为零扩展形式。

指令中指定的任何条件必须与条件代码状态匹配，否则不执行指令。

数据地址由“英特尔 XScale(R)”核心生成，其中 address_mode 是协处理器寄存器的加载与存储使用的地址模式 5。此指令将寄存器 Rn 用作基址与偏移量。地址计算中使用的偏移量总是以字节为单位指定给汇编器。不过，对于 W 与 D 限定符，偏移量必须是字的倍数（即，可以被 4 整除）。此指令允许使用字与双字加载变体，其偏移量范围 (0 - 0x3FC) 大于字节与半字等加载变体的偏移量范围 (0 - 0xFF)。请参阅下表。

异常

无

影响的 SIMD 标志

不影响 SIMD PSR 标志 (wCASF) 或“饱和”标志 (wCSSF)。

操作

if (B specified) then

dest[7:0] = Mem[<address_mode>] ;

dest[63:8]=0;

else if (H specified) then

dest[15:0] = Mem[<address_mode>] ;

dest[63:16]=0;

else if (W specified) then

dest[31:0] = Mem[<address_mode>] ;

dest[63:32]=0;

else if (D Specifed) then

dest[63:0] = Mem[<address_mode>] ;
