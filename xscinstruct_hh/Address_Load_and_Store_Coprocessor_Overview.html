加载与存储协处理器概述

“加载与存储协处理器”寻址模式分成 4 个子模式。它们用于计算加载与存储协处理器指令的地址。此寻址模式的一般语法在后续部分提供。

一般语法

opcode [condition] [L] cpname, cpdest, addr_mode

其中：

 opcode

指定指令操作。

 condition

16 个条件之一。请参阅条件代码状态。

 L

指定位 [22]、N 位：
如果 L 设置为 1，则 N 位置为 1，并指定长加载或长存储。如果 L 未置位，则 N 位是零，并指定短加载或短存储。如需有关详细信息，请参阅 LDC/LDC2 指令与 STC/STC2 指令。

 cpname

协处理器名称 (p0..p15)

 cpdest

指定目标协处理器寄存器。

 

addr_mode 是以下 6 种子模式之一：

子模式

说明

 [basereg, #+/-value*4]

偏移量选项：
创建地址序列。第一个地址是将 8 位立即数偏移量乘以四后加到基址寄存器，或从中减去。后续的地址由前一个地址递增四得到。请参阅加载与存储协处理器：立即数偏移量 (AM 5)。

 [basereg #+/-value*4]!

事先变址选项：
创建地址序列。第一个地址是将 8 位立即数偏移量乘以四后加到基址寄存器，或从中减去。如果传递了某个条件，则将值写回基址寄存器。后续的地址由前一个地址递增四得到。请参阅加载与存储协处理器：立即数事先变址 (AM 5)。

 [basereg], #+/-value*4

事后变址选项：
创建地址序列。如果传递某个条件，则第一个地址是将 8 位立即数偏移量乘以四后加到基址寄存器，或从中减去。然后将值写回基址寄存器。
后续的地址由前一个地址递增四得到。请参阅加载与存储协处理器：立即数事后变址 (AM 5)。

 [basereg], option

非变址选项：
创建地址序列。第一个地址是基址寄存器 basereg 的值。后续的地址由前一个地址递增四得到。基址寄存器不更新，因此各个位可以由 option 用于指定其它协处理器选项。请参阅加载与存储协处理器：非变址 (AM 5)。

 value

其中：

 P

P == 0 表示事后变址寻址。基址寄存器的值用作内存地址。偏移量应用于基址寄存器的值，并写回基址寄存器。
P == 1 表示偏移量寻址或事先变址寻址。（这由 W 位确定）。内存地址是通过将偏移量应用于基址寄存器值而生成的。

 U

表示是将偏移量加到基址还是从中减去：
如果 U == 1，则偏移量加到基址上。
如果 U == 0，则偏移量从基址中减去。

 N

此位有依赖于协处理器的含义。建议使用此位区分要传输的不同大小的值。

 W

有两个含义：
P == 0：
如果 W == 0，则执行非变址寻址。为此 U 位必须是 U == 1，否则结果未定义或无法预知。
如果 W == 1，则执行事后变址寻址。
P == 1：
如果 W == 0，则基址寄存器不更新（偏移量寻址）。
如果 W == 1，计算的内存地址将写回基址寄存器（事先变址寻址）。

 L

表示加载或存储：
L==1 表示加载。
L==0 表示存储。

 cpnum

协处理器编号

 

W 位的含义取决于 P 的设置：

 

 P==0

 P==1

 W==0

执行非变址寻址。为此 U 位必须是 U == 1，否则结果未定义或无法预知。

基址寄存器不更新（偏移量寻址）。

 W==1

执行事后变址寻址。

计算的内存地址写回基址寄存器（事先变址寻址）。
