非变址 (AM 5)

语法

[basereg], option

其中：

 basereg

指定包含基址的寄存器。

 option

指定其它的协处理器选项。此选项作为 0-255 范围内的整数在指令语法中指定，并且使用 { 与 } 括起来。

说明

寻址模式“未变址”执行以下操作（伪代码说明）：

if ConditionPassed (condition) then
start_addr = basereg
addr = start_addr
while (NotFinished (coprocessor[cpnum]))
addr = addr + 4
end_addr = addr

此寻址模式创建连续地址序列。第一个地址是基址寄存器 basereg 的值。后续地址由前一个地址递增四得到。这会持续执行到协处理器发出指令结束信号。这样，协处理器便可以访问协处理器定义的数据。

基址寄存器 basereg 不更新。因此，基于“英特尔 XScale(R)”微体系结构的处理器不使用位 [7:0]。这些位可用于给协处理器指定其它的指令选项。

协处理器不得要求进行超过 16 个字的传输。

备注

N 位依赖于协处理器。

L 位表示“加载”( L == 1 ) 或“存储”( L == 0 ) 指令。
如果 R15 指定为寄存器 basereg，则使用的值是当前指令的地址加 8。

注意

如果 U 位（位 [23]）未置位，则结果未定义或将会无法预知。

备注

在这种寻址模式中，基于“英特尔 XScale(R)”微体系结构的处理器不使用选项位（位 [7:0]）。这些位可用于给协处理器指定其它的指令选项。

示例

LDC2    P0, CR15, [R3]
