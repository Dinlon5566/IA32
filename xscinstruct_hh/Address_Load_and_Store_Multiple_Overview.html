多重加载与存储概述

“多重加载与存储”寻址模式分成 4 个子模式。加载指令从内存加载 reg 指定的部分、甚至是全部通用寄存器。存储指令将 reg 指定的部分、甚至是全部通用寄存器存储到内存。此寻址模式产生地址是有顺序的；因此，reg 指定的寄存器按照升序从自低到最的内存地址进行加载。此寻址模式的一般语法在后续部分提供。

一般语法

LDM|STM [condition] addr_mode basereg[!], reg {,reg}

其中：

 condition

16 个条件之一。请参阅条件代码状态。

 basereg

addr_mode 使用的基址寄存器。

 !

设置 W 位 (21)。

 reg

要加载的有效寄存器。

addr_mode 是以下 4 种子模式之一：

它形成“多重加载与存储”指令的地址范围。请参阅多重加载/存储：之后递增 (AM 4)。

之前递增。
它形成“多重加载与存储”指令的地址范围。请参阅多重加载/存储：之前递增 (AM 4)。

它形成“多重加载与存储”指令的地址范围。请参阅多重加载/存储：之前递减 (AM 4)。

其中：

位 [24]

是 P 位。请参阅下文。

位 [23]

是 U 位。
U == 1 表示从 basereg 向上传输。
U == 0 表示从 basereg 向下传输。

位 [22]

是 S 位。
S 位表示，对于加载 PC 的“加载”指令，CPSR 是从 SPSR 加载的。在特权模式中，它表示，对于不加载 PC 的“加载”指令以及所有的“存储”指令，传输“用户”模式库寄存器。

位 [21]

是 W 位。
W 位表示 basereg 进行更新。
U == 1 表示 basereg 按 reglist 中寄存器数量的四倍递增。
U == 0 表示 basereg 按 reglist 中寄存器数量的四倍递减。

位 [20]

是 L 位。
L == 1 表示“加载”指令。
L == 0 表示“存储”指令。

位 [15:0]

 reglist 存在一个位专用于每个寄存器：位[0] 表示 R0，位[15] 表示 R15 ( PC )。至少应设置一个位。

 

P 位（位 [24]）的含义取决于 U 位的设置：

 

 U==0

 U==1

 P==0

basereg 中包含的地址超出范围顶部一个字。

basereg 中包含的地址低于范围底部一个字。

 P==1

basereg 中包含的地址在范围底部访问。

basereg 中包含的地址在范围顶部 (U == 1) 访问。

替代性名称

对于指令块传输，IA、IB、DA 及 DB 寻址模式最为有用，特别是“多重加载与存储”指令使用寻址模式时，数据加载与存储使用同样的方式。
“多重加载与存储”指令用于访问堆栈时，弹出与压入操作按照相反的方向调整堆栈，此时便需要这些不同的寻址模式。
以下是提供的堆栈操作替代性语法：

堆栈

语法

说明

空白升序

EA

堆栈指针指向第一个未使用的位置，并向内存位置增加的方向移动。

完全升序

FA

堆栈指针指向使用的上一个位置，并向内存位置增加的方向移动。

空白降序

ED

堆栈指针指向未使用的第一个位置，并向内存位置减小的方向移动。

完全降序

FD

堆栈指针指向使用的上一个位置，并向内存位置减小的方向移动。

定义协处理器堆栈时，建议使用 EA 或 FD 堆栈类型。对于此类型的堆栈，协处理器数据可以分别使用一条 STC 或 LDC 指令压入或弹出。ED 与 FA 协处理器堆栈访问需要使用多指令序列。
