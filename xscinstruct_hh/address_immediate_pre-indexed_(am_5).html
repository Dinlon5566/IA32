立即数事先变址 (AM 5)

语法

[basereg, #+/-value*4]!

其中：

 basereg

指定包含基址的寄存器。

 value

指定 8 位立即数偏移量，乘以 4 后与 basereg 的值一起使用，以形成地址。

说明

寻址模式“立即数事先变址”执行以下操作（伪代码说明）：

if ConditionPassed (condition) then
if U == 1 then
  basereg = basereg + value * 4
else /* U == 0 */
  basereg = basereg - value * 4
start_addr = basereg
addr = start_addr
while (NotFinished (coprocessor[cpnum]))
  addr = addr + 4
end_addr = addr

此寻址模式创建连续地址序列。对于第一个地址，将立即数偏移量乘以四后加到基址寄存器 basereg 的值，或从中减去。如果指令中指定条件的与条件代码状态匹配，则此地址写回基址寄存器 basereg。后续地址由前一个地址递增四得到。这会持续执行到协处理器发出指令结束信号。这样，协处理器便可以访问协处理器定义的数据。

协处理器不得要求进行超过 16 个字的传输。

 备注

N 位依赖于协处理器。
L 位表示“加载”( L == 1 ) 或“存储”( L == 0 ) 指令。

 注意

如果 R15 指定为寄存器 basereg，则结果将会无法预知。

示例

LDCL    P5, CR12, [R8, #4]!
