寄存器偏移量 (AM 3)

语法

[basereg, +/-indexreg]

其中：

 basereg

指定包含基址的寄存器。

 indexreg

指定包含可加到 basereg 或从中减去的值的寄存器。

说明

寻址模式“寄存器偏移量”执行以下操作（伪代码说明）：

if U == 1 then
addr = basereg + indexreg
else /* U == 0 */
addr = basereg - indexreg

变址寄存器 indexreg 的值加到基址寄存器 basereg 的值，或是从中减去。

对于指针加偏移量算术，以及对于访问数组的单个元素，此寻址模式非常有用。

L 位表示“加载”( L == 1 ) 或“存储”( L == 0 ) 指令。

S 位表示有符号 ( S == 1 ) 或无符号 ( S == 0 ) 半字访问。

H 位表示半字 ( H == 1 ) 或有符号字节 ( H == 0 ) 访问。

 备注

如果 R15 指定为寄存器 basereg，则使用的值是当前指令的地址加 8。
如果 S 与 H 表示无符号字节（ S == 0 且 H == 0），则指令不使用此寻址模式。这是一个乘法指令，即 SWP 或 SWPB 指令，或是算术或加载/存储指令扩展区中未分配的指令。
无符号字节由指令 LDRB、LDRBT、STRB 及 STRBT 访问。这些指令使用寻址模式 2。如需有关详细信息，请参阅“寻址模式 2：加载与存储字或无符号字节”。
如果 S 与 L 表示有符号存储指令（S == 1 且 L== 0），则此指令是加载/存储扩展区中未分配的指令。有符号字节与半字使用指令 STRB 与 STRH 存储。

 注意

如果 R15 指定为寄存器 indexreg，则结果将会无法预知。

示例

LDRSH    R4, [R2, R5]
