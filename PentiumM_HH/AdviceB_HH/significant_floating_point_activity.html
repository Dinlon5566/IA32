浮点操作活动明显

浮点操作活动明显表示可能适用专门针对浮点算法的优化方法。

在以下情况中，此细节具有实质性意义：
计数器的商（失效的浮点操作/失效的指令）较高。值 0.00 应视为较低，值 0.95 应视为较高。

建议

    • 考虑使用压缩版的“数据流单指令多数据扩展指令集”(SSE) 或 SSE2 代码。由于使用标量 SSE 代码而不使用常规浮点代码可能会降低性能（具体取决于指令混合情况），因此请使用编译器的 /QxW 标志检查性能影响。原因在于，熔合代码跟 x87 代码、标量 SSE 代码不同，它无法进行从内存到寄存器的操作。通常情况下，使用压缩代码极有可能改善性能。

    • 研究占用大部分执行时间的循环嵌套的汇编代码，以了解编译器执行了哪些类型的转换，并考虑还可能执行其它哪些优化。

    • 考虑将长延迟浮点算术运算替换成更快的运算。
      设法将长延迟浮点算术运算（如除法）替换成更快的运算（如乘以倒数）。如果要频繁执行除法，除非确有必要，否则请不要使用较高的精度。与双精度（64 位）或扩展双精度（80 位）相比，单精度（32 位）的除法速度更快，并且只消耗一半的内存空间。请注意，更改数据类型（例如，从 double 更改成 float）不会自动更改执行算术运算时的精度。精度可通过更改浮点状态寄存器进行设置。降低除法开销的另一种办法是使用 SSE 标量或压缩除法指令。如需有关详细信息，请参阅处理器的“程序员参考手册”。

    • 如果代码需要执行 float 到 int 转换，请确保使用 float 到 int 快速转换例程。许多函数库所做的工作都超过必需做的工作，请确保 float 到 int 的转换例程速度较快。请注意，由于 FORTRAN 与 C 语言的语义方面的原因，float 到 int 的转换可能会暗中发生，因此可能不容易注意到发生了 float 到 int 的转换。有时候，通过查看“VTune™ 性能分析器”的输出中是否出现例程 _ftol()，可以检测出是否发生 float 到 int 转换。对于某些编译器（如英特尔(R) C++ 编译器），由于例程是内嵌的，因此更难检测此种转换。使用 SSE 指令可以解决此问题，SSE 指令使用截断转换指令。  

    • 确保应用程序中的数据在正确的范围内。数值超出范围会导致非常高的开销。如果使用“数据流单指令多数据扩展指令集”(SSE)，并且在处理浮点值，请尝试使用 SSE 的清除为零模式。

    • 尽可能使用“英特尔(R) MKL”（数学内核库）。这些库包含许多常用的线性代数例程，如 BLAS3、LAPACK 等等。

    • 考虑编译器驱动的矢量化。如需有关编译器驱动的矢量化的详细信息，请参阅编译器文档。目前，对于英特尔(R) C++ 编译器，矢量化功能通过命令行开关 -QxW 激活。
