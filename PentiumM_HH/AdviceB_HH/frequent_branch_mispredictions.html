频繁发生分支预测失误

改善分支的可预测性可能会给代码的性能带来明显的影响。

在以下情况中，此细节具有实质性意义：

计数器的比率（失效的分支预测失误/失效的微操作）不佳。值 0.005 应视为良好，值 0.025 应视为不佳。

建议

  • 考虑使用“英特尔(R) 编译器”的“档案导引基本指令块优化”(PGBO) 选项重新调整基本指令块结构，使产生的分支更少。

  • 考虑使用“英特尔编译器”选项 -Qxi，将分支指令替换成 cmov 与 fcmov 指令。

  • 设计代码以改善分支可预测性。

    • 确保每条 CALL 指令都有匹配的 return。

    • 避免混用数据与指令。

    • 展开非常短的循环。

    • 分支基于随机预测的数据时，请考虑使用条件 MOV 指令、“数据流单指令多数据扩展指令集”(SSE) 或“MMX(TM) 技术”指令修改代码以避免分支。
      例如，将以下代码：

if (a>0)

c=x

else

c=y

 

更改成以下版本：

int val [2]={x,y};

c=val[a>0];

  • 在汇编代码中，可以考虑将分支指令替换成 cmov、fcmov 及 setcc 指令，从而消除它们。此外在有些情况下，您还可以使用 SIMD 比较（创建掩码）与逻辑指令（与/或/非），以布尔运算执行操作。如需查看示例，请参阅“MMX(TM) 技术概述”文档中的“色度键控”部分，此文档可以在 http://developer.intel.com 上找到。
